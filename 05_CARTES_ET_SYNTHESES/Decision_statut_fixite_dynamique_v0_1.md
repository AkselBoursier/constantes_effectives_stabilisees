# Décision sur le statut de la fixité dynamique v0.1

## 0. Statut

```text
statut : décision proposée par l’audit, soumise à validation finale ;
fonction : empêcher l’exportation du cycle 10 en loi ou classe générale ;
appuis : audit des trajectoires, carte des voisinages, résultats des phases 2 à 4.
```

## 1. Décision principale

```text
La fixité dynamique est abandonnée comme principe général, loi, forme universelle
ou classe transversale des constantes.
```

Elle est conservée dans deux usages limités :

1. comme nom généalogique du cycle 10 ;
2. comme désignation locale d’une quasi-fixité produite ou maintenue par une
dynamique explicitement instruite.

## 2. Motivation

Toute attribution dynamique dépend au minimum de :

```text
un porteur ;
des équations ;
un régime ;
une fenêtre temporelle ;
une tolérance ;
un ensemble de conditions ;
un bilan énergétique ;
un critère de rupture.
```

Sans ces éléments, `fixité dynamique` ne distingue ni stabilité, ni
métastabilité, ni freeze-out, ni amortissement, ni validité effective.

## 3. Ce qui est rejeté

Les formulations suivantes ne sont plus autorisées comme énoncés transversaux :

```text
les constantes sont des fixités dynamiques ;
les lois produisent leurs propres constantes ;
la dynamique stabilise généralement les paramètres ;
la nature converge vers des valeurs fixes ;
la fixité dynamique constitue une loi locale de la physique.
```

Elles peuvent être mentionnées comme hypothèses historiques ou formulations
abandonnées, avec leur statut.

## 4. Ce qui reste autorisé

```text
Dans le modèle M, le champ X atteint une quasi-fixité sur la fenêtre Delta t et à
la tolérance epsilon.

Le mécanisme Y possède un bassin de trajectoires conduisant à un régime tardif
faiblement variable.

La phase 4 du cycle 10 ouvre une fenêtre phénoménologique de dissipation, sans
réalisation microscopique acquise.

Le projet compare plusieurs profils temporels sans les convertir en une classe
physique unique.
```

## 5. Remplacement conceptuel

Le terme transversal recommandé est :

```text
profils temporels de stabilisation des fixités
```

Il désigne un programme de comparaison, non une catégorie naturelle déjà établie.
Il doit toujours être accompagné du profil technique précis :

```text
attraction ;
relaxation ;
métastabilité ;
suivi adiabatique ;
freeze-out ;
dissipation ;
transition ;
quasi-stationnarité ;
autre mécanisme nommé.
```

## 6. Effet sur le cycle 10

### Conservé

- les cadrages sur ce qui varie ;
- la séparation RG / thermique / dynamique / inférentielle ;
- les quatre phases déjà réalisées ;
- les résultats numériques reproduits ;
- les échecs des modèles conservatifs simples ;
- la fenêtre dissipative phénoménologique ;
- la dette microscopique explicite.

### Déplacé

Le cycle devient :

> **un test local de quasi-fixité électrofaible dans un modèle cosmologique
> simplifié**.

Il ne fournit plus directement :

- un concept général pour toutes les constantes ;
- une preuve que la fixité électrofaible réelle est dynamiquement constituée ;
- une réponse générale au fine-tuning ;
- une loi de formation des régimes.

## 7. Condition de poursuite microscopique

La réalisation microscopique du taux dissipatif reste utile seulement si elle
permet d’évaluer au moins :

```text
l’origine du canal ;
sa dépendance en température et en champ ;
les produits de dissipation ;
la rétroaction ;
la compatibilité avec les contraintes ;
le déplacement éventuel du réglage.
```

Son objectif n’est plus de sauver un concept général. Il est de conclure proprement
sur un cas physique local.

## 8. Effet sur la méthode v1.5

La passe `constitution ou conditions d’opérativité` reste valide. Elle doit être
lue comme une question ouverte dont les sorties incluent :

```text
mécanisme dynamique ;
architecture non dynamique ;
validité de régime ;
relation formelle ;
opération métrologique ;
paramètre libre ;
absence actuelle d’explication.
```

La décision renforce donc la méthode v1.5 au lieu de l’invalider.

## 9. Effet sur les futurs documents

Le premier dégagement v0.3 devra :

1. ne pas présenter la fixité dynamique comme pilier général ;
2. conserver le cycle 10 comme étude de cas et test de limites ;
3. distinguer les registres temporel, RG, métrologique et documentaire ;
4. présenter les profils temporels comme programme comparatif sous condition ;
5. reconnaître explicitement que la littérature spécialisée possède déjà les
concepts physiques locaux.

## 10. Formule de clôture

> La fixité dynamique n’est pas une loi cachée du projet. Elle devient un cas
> local, instructif précisément parce qu’il oblige à spécifier le temps, le régime,
> l’énergie et la rupture.
