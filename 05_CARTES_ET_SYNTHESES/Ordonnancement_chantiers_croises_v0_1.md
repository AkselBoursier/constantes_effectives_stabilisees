# Ordonnancement des chantiers croises v0.1

## 0. Statut documentaire

```text
statut : note de pilotage ;
fonction : ordonner les prochains chantiers apres l'audit terminologique actif ;
portee : cycles, architectures, limites, echelles philosophiques,
livrables et raccords transversaux.
```

Cette note produit un ordre de dependance entre les chantiers.

Elle sert a choisir l'ordre des reprises afin que chaque chantier renforce
le suivant dans une progression explicite.

## 1. Regle d'ordonnancement

Un chantier doit passer plus tot lorsqu'il remplit au moins une des fonctions
suivantes :

```text
1. stabiliser un vocabulaire commun ;
2. clarifier le centre de gravite d'un cycle ;
3. prevenir une interpretation trop forte ;
4. rendre un livrable possible a grain resserre ;
5. reduire une ambiguite qui se propage dans plusieurs documents.
```

Le critere principal est la capacite d'un chantier a rendre les autres
chantiers plus justes, plus lisibles et moins ambigus.

## 3. Pourquoi commencer par les cycles et architectures

Les cycles portent les centres de gravite du corpus.

S'ils sont mal equilibres, les autres chantiers risquent de reprendre une
structure deformee :

```text
Saveur-Higgs : constitution ;
effective basse energie : validite de regime ;
cosmologie : inference et reconstruction ;
neutrinos : bornes et acces heterogenes ;
seuil electrofaible : entree de regime et jonction BEH.
```

Ce chantier sert donc a verifier que chaque cycle conserve sa fonction propre
et que les architectures les articulent selon leurs centres de gravite.

Il doit aussi controler les effets de deplacement produits par les reprises
recentes :

```text
1. poids excessif possible du SI ;
2. ambiguite autour de "situe" ;
3. retour discret de "famille" ou de "relation" comme premiers noms ;
4. risque de lecture epistemologique derealisante ;
5. statut encore sensible du seuil electrofaible et de BEH.
```

## 4. Ce que chaque chantier rend possible

### 4.1. Audit cycles et architectures

Il rend possible :

```text
- une lecture non juxtaposee des cycles ;
- une verification des centres de gravite ;
- une correction des architectures actives ;
- une base stable pour traiter les limites.
```

Sortie attendue :

```text
matrice courte des cycles, de leurs fonctions, de leurs dependances
et des corrections documentaires a appliquer.
```

### 4.2. Audit des regimes de limite

Il rend possible :

```text
- distinguer borne, tension, non-detection, substitution et stress test ;
- maintenir une tension a son rang d'escalade ;
- proteger le degagement de these contre une formulation trop forte ;
- reprendre cosmologie et neutrinos avec une prudence plus lisible.
```

Sortie attendue :

```text
typologie operative des limites et liste des formulations a resserrer.
```

### 4.3. Audit philosophie / livrables

Il rend possible :

```text
- separer instrument, voisinage, hypothese et argument ;
- maintenir chaque element philosophique a son rang probatoire ;
- identifier ce qui peut entrer dans un premier livrable ;
- conserver les ouvertures comme reserves actives.
```

Sortie attendue :

```text
table de tri entre acquis mobilisables, horizons de travail
et formulations a tenir en reserve.
```

### 4.4. Ouverture du pan philosophique

Elle rend possible :

```text
- distinguer une fenetre active du pan philosophique complet ;
- ouvrir les foyers de description ontologique prudente, epistemologie des
  acces, realisme medie, coordination, modalite, historicite, constitution,
  limites, architecture et institution ;
- explorer les voisinages au-dela de leur question d'entree ;
- garder le choix du livrable comme effet d'une exploration plus large.
```

Sortie attendue :

```text
cadrage d'ouverture du pan philosophique ;
carte des foyers philosophiques du projet.
```

### 4.5. Extraction epistemologique depuis les cycles physiques

Elle rend possible :

```text
- partir des cycles physiques plutot que des themes historiques ;
- distinguer regime d'effectuation, regime d'acces, qualification du statut
  et positionnement philosophique ;
- degager le noyau epistemologique du corpus : realisme medie, acces,
  validite, definition, reconstruction, bornes et architecture ;
- donner aux situations philosophiques un rang aval de resonance et de
  contraste.
```

Sortie attendue :

```text
extraction des axes epistemologiques depuis les cycles physiques ;
requalification de la table des situations comme resonance aval ;
preparation d'une fiche sur les axes epistemologiques structurants.
```

### 4.6. Instruction des echelles philosophiques

Elle rend possible :

```text
- mesurer le potentiel philosophique du corpus avant le choix du livrable ;
- distinguer qualification, acces, realisme medie, formation, modalite,
  historicite, metrologie, architecture et exportabilite ;
- comparer les echelles selon leur maturite, leur originalite et leurs dettes
  de lecture ;
- choisir le livrable a partir d'une cartographie plus large que le seul axe
  actuellement le mieux stabilise.
```

Sortie attendue :

```text
extraction epistemologique depuis les cycles physiques ;
table des echelles et des situations philosophiques comme resonance ;
fiches courtes par lots d'echelles ;
synthese de maturite avant choix du livrable.
```

### 4.7. Plan de livrable resserre

Il rend possible :

```text
- choisir le premier texte defendable apres instruction philosophique ;
- limiter l'exposition de la machinerie interne ;
- ordonner les plans A1, A2, A3, A4 et B ;
- transformer le corpus en trajectoire de publication progressive.
```

Sortie attendue :

```text
plan court du premier livrable et liste des prealables indispensables.
```

## 5. Chantiers transversaux a surveiller

Ces points fonctionnent comme controles transversaux dans les audits ci-dessus :

```text
- definition de la constante effective stabilisee ;
- statut de "qualifiee" et de "situee" ;
- place du SI comme cas local plutot que modele general ;
- rapport entre acces epistemique et realite des processus ;
- seuil electrofaible et BEH comme zone de jonction ;
- distinction entre cycle, chantier, architecture et livrable ;
- articulation entre question directrice initiale et theses derivees.
- ouverture du pan philosophique avant extraction epistemologique.
- extraction epistemologique depuis les cycles physiques avant table des
  situations.
- pluralite des echelles philosophiques avant choix du livrable.
```

## 7. Regle de travail associee

La reprise reste selective.

Elle doit produire :

```text
1. un diagnostic court ;
2. une matrice de dependances ;
3. une liste de corrections ciblees ;
4. seulement ensuite les modifications documentaires necessaires.
```

Conformement au protocole GitHub ajuste, cette note peut rester non versionnee
jusqu'a la fin du prochain cycle coherent de travail, sauf besoin de
sauvegarde explicite.

## 8. Formule de cloture

```text
L'ordre des chantiers doit suivre la dependance intellectuelle :
d'abord stabiliser les centres,
puis qualifier les limites,
puis trier les ouvertures,
puis ouvrir le pan philosophique comme champ,
puis extraire les axes epistemologiques depuis les cycles physiques,
puis instruire les echelles philosophiques,
puis seulement resserrer le livrable.
```
