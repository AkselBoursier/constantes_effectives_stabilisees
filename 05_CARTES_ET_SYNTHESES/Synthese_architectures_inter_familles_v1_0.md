# Synthese des architectures actives v1.0

## Invariants methodologiques, differences internes et statut de rang

### 1. Fonction de la synthese

Cette synthese compare les quatre architectures actives stabilisees par la carte consolidee v1.1.

Le nom de fichier conserve `inter_familles` pour la genealogie du corpus.

Dans les formulations actives, le niveau vise est celui de la solidarisation
de fonctions, d'operations, d'acces et de limites dans un regime.

Architectures traitees :

```text
Saveur-Higgs
Metrologique SI
Effective basse energie
Cosmologique
```

Objectif :

```text
1. identifier ce que ces architectures ont en commun ;
2. distinguer leurs formes internes ;
3. fixer les erreurs de rang qu'elles permettent d'eviter ;
4. preparer une eventuelle mise a jour methodologique v1.2.
```

La synthese ne remplace pas les notes d'architecture. Elle en extrait les invariants.

### 2. Resultat central

Les quatre cas confirment la meme regle :

```text
Une architecture n'ajoute pas une famille.
Elle montre comment plusieurs familles deviennent solidaires dans un regime de description.
```

Resultat de rang :

```text
architecture active = niveau methodologique autonome de solidarisation
```

mais non :

```text
sortie taxonomique superieure
```

La difference est decisive :

```text
Famille    : que fait principalement une grandeur ?
Architecture : comment plusieurs fonctions deviennent-elles solidaires ?
```

### 3. Les quatre architectures stabilisees

| Architecture | Forme dominante | Fonctions et roles mobilises | Corpus de reference |
|---|---|---|---|
| Saveur-Higgs | Constitutive | Echelle, Couplage, Orientation, roles spectraux, relations qualifiees | v, Y_f, m_f, CKM, PMNS, Delta m^2 |
| Metrologique SI | Definitionnelle | Convention, composition exacte, Echelle, relation qualifiee | k_B, N_A, R, c, h, e, K_J, R_K |
| Effective basse energie | Effective / validite | Raccordement, Echelle, Couplage, Validite, liaisons de regime qualifiees | G_F, M_W, alpha_s(Q^2), Lambda_QCD, alpha_G(E), M_Pl |
| Cosmologique | Inferentielle / reconstructive | Fonctions architecturales, roles relationnels, acces | Lambda, H_0, Omega_i, w, A_s, n_s, sigma_8, S_8 |

Chaque architecture est robuste parce qu'elle organise plusieurs fonctions.

Aucune n'est robuste comme famille unique.

### 4. Architecture Saveur-Higgs

Forme dominante :

```text
architecture constitutive
```

Elle articule :

```text
Echelle
+ Couplage
+ Relation constitutive sous audit
+ Orientation
```

Structure :

```text
v          : echelle electrofaible
Y_f        : couplages de texture
m_f        : relations constitutives de regime
CKM / PMNS : orientations de saveur
Delta m^2  : relations spectrales
```

Invariance methodologique :

```text
une masse, un couplage et une orientation ne suffisent pas seuls a decrire le secteur.
```

Le secteur devient intelligible lorsque les fonctions sont articulees.

Risque controle :

```text
fragmenter le secteur en familles locales trop nombreuses
```

ou, inversement :

```text
reduire Saveur-Higgs a une sortie taxonomique locale.
```

Decision :

```text
Saveur-Higgs = architecture constitutive de solidarisation.
```

### 5. Architecture metrologique SI

Forme dominante :

```text
architecture definitionnelle
```

Elle articule :

```text
Convention
+ Composition exacte
+ Echelle physique conservee
+ Realisation pratique
+ Systeme coherent d'unites
```

Structure :

```text
k_B, N_A, c, h, e : constantes definissantes primaires
R, K_J, R_K       : compositions exactes
```

Invariance methodologique :

```text
exactitude conventionnelle n'est pas precision empirique.
```

Une valeur peut etre exacte dans le SI sans que son role physique soit epuise.

Risque controle :

```text
reduire c, h ou e a de simples conventions
```

ou, inversement :

```text
confondre exactitude SI et mesure parfaite.
```

Decision :

```text
Metrologique SI = architecture definitionnelle de solidarisation.
```

### 6. Architecture effective basse energie

Forme dominante :

```text
architecture de validite
```

Elle articule :

```text
Raccordement
+ Echelle de validite
+ Couplage de regime
+ Fonctions Validite / Seuil / Hierarchie
```

Structure :

```text
G_F / M_W                 : integration d'un mediateur massif
alpha_s(Q^2) / Lambda_QCD : flot de couplage et changement de regime
alpha_G(E) / M_Pl         : couplage dimensionne et suppression d'echelle
```

Invariance methodologique :

```text
une description effective vaut dans un domaine, pas absolument.
```

Risque controle :

```text
transformer Effectif en sortie taxonomique superieure
```

ou :

```text
oublier le domaine de validite.
```

Decision :

```text
Effective basse energie = architecture de validite de description.
```

### 7. Architecture cosmologique

Forme dominante :

```text
architecture inferentielle et reconstructive
```

Elle articule :

```text
Fond
+ Parametre d'etat
+ Densite normalisee
+ Relation d'etat
+ Condition initiale
+ Observable reconstruite
+ Regimes d'acces
+ Dependances de modele
+ Tensions de donnees
```

Structure :

```text
Lambda        : terme de fond
H_0           : parametre d'etat
Omega_i       : densite normalisee
w             : relation d'etat
A_s / n_s     : conditions initiales
sigma_8 / S_8 : observables reconstruites
```

Invariance methodologique :

```text
une grandeur cosmologique se classe par role, modele et acces.
```

Risque controle :

```text
transformer la cosmologie en famille
```

ou :

```text
transformer une tension d'acces en verdict physique.
```

Decision :

```text
Cosmologique = architecture inferentielle et reconstructive.
```

### 8. Quatre formes d'architecture

La comparaison permet de distinguer quatre formes.

| Forme | Architecture | Question directrice |
|---|---|---|
| Constitutive | Saveur-Higgs | Comment des masses, couplages, relations et orientations constituent-ils un secteur ? |
| Definitionnelle | Metrologique SI | Comment des constantes fixent-elles un systeme d'unites sans epuiser leur role physique ? |
| Validite | Effective basse energie | Dans quel domaine une description reste-t-elle applicable ? |
| Inferentielle / reconstructive | Cosmologique | Comment un reseau de grandeurs devient-il stabilise par modele et acces ? |

Ces formes ne sont pas des familles.

Elles qualifient le type d'architecture.

### 9. Invariants transversaux

Les quatre architectures partagent huit invariants.

#### 9.1 Solidarisation de fonctions

Une architecture rend solidaires plusieurs familles ou fonctions.

Elle n'est pas la somme brute de ses grandeurs.

Regle :

```text
architecture = solidarisation fonctionnelle dans un regime
```

#### 9.2 Correction de rang

Chaque architecture decrit positivement une solidarisation qui depasse une sortie taxonomique locale.

| Tentation | Correction |
|---|---|
| lecture Saveur-Higgs comme famille | architecture constitutive |
| lecture Metrologie comme famille | architecture definitionnelle |
| lecture Effectif comme famille | architecture de validite |
| lecture Cosmologie comme famille | architecture inferentielle |

#### 9.3 Regime avant essence

Une grandeur ne se classe pas hors regime.

Exemples :

```text
h : Convention dans le SI, Echelle d'action dans le regime quantique.
e : Convention dans le SI, charge dimensionnee ; alpha reste le couplage sans dimension.
H_0 : parametre d'etat cosmologique, non constante fondamentale.
M_W : Echelle avec fonction de validite pour la theorie de Fermi.
```

#### 9.4 Regime d'acces obligatoire

Une architecture impose de dire comment les grandeurs sont connues.

Exemples :

```text
SI : realisations pratiques.
Effective basse energie : domaines de validite et regimes experimentaux.
Cosmologie : CMB, BAO, distance ladder, lentille faible, modeles d'inference.
Saveur-Higgs : mesures de masses, melanges, oscillations, ajustements.
```

#### 9.5 Limites explicites

Chaque architecture est definie autant par ce qu'elle ne dit pas que par ce qu'elle articule.

```text
SI ne donne pas l'origine physique de c, h ou e.
Effective basse energie ne donne pas toujours la completion.
Cosmologie ne resout pas les tensions H_0 ou S_8.
Saveur-Higgs ne donne pas l'origine ultime des hierarchies de saveur.
```

#### 9.6 Stabilisation situee

La robustesse d'une grandeur se qualifie toujours dans un regime,
par des acces explicites et avec des limites conservees.

Types de stabilisation :

```text
empirique
conventionnelle
composition exacte
reconstruite
modele-dependante
bornee
en tension entre acces
```

#### 9.7 Non-absorption

Une architecture ne doit pas absorber toute grandeur voisine.

Exemples :

```text
Toute constante dimensionnee n'est pas SI.
Toute approximation n'est pas effective.
Toute grandeur astrophysique n'est pas cosmologique.
Toute matrice ou masse n'est pas automatiquement Saveur-Higgs.
```

#### 9.8 Matrice commune

Chaque architecture peut etre decrite par les memes questions minimales :

```text
1. Quel corpus ?
2. Quelles familles principales ?
3. Quelles fonctions transversales ?
4. Quelles fonctions architecturales ?
5. Quel regime physique ?
6. Quel regime d'acces ?
7. Quel mode de stabilisation ?
8. Quelle erreur de rang evitee ?
9. Quelle limite explicite ?
```

### 10. Differences structurantes

Les architectures different par leur mode de cohesion.

| Architecture | Mode de cohesion | Stabilisation dominante | Risque principal |
|---|---|---|---|
| Saveur-Higgs | Constitution d'un secteur | Empirique et relationnelle | Sur-fragmentation ou famille artificielle |
| Metrologique SI | Definition d'un systeme d'unites | Conventionnelle et par composition exacte | Confondre exactitude et physique complete |
| Effective basse energie | Validite d'une description limitee | Empirique, theorique et de regime | Oublier la limite de validite |
| Cosmologique | Reseau d'inference et reconstruction | Modele-dependante et pluralite d'acces | Confondre tension et verdict physique |

Cette table montre que les architectures ont le meme rang methodologique, mais pas la meme forme.

### 11. Statut des fonctions transversales

Les fonctions transversales apparaissent differemment selon l'architecture.

| Fonction transversale | Architecture ou elle est la plus visible | Role |
|---|---|---|
| Seuil | Effective basse energie | Passage ou limite de regime |
| Validite | Effective basse energie | Domaine d'application d'une description |
| Hierarchie | Saveur-Higgs, effective basse energie | Ecarts de masses, couplages ou echelles |
| Dephasage | Saveur-Higgs | Manifestation dynamique d'une relation spectrale |
| Borne | Cosmologique, neutrinos | Acces par limite plutot que mesure positive |

Regle :

```text
une fonction transversale peut etre centrale dans une architecture sans devenir famille.
```

### 12. Statut des fonctions architecturales

Les fonctions architecturales sont surtout visibles en cosmologie, mais le principe est general.

Exemple cosmologique :

```text
Fond
Parametre d'etat
Densite normalisee
Relation d'etat
Condition initiale
Observable reconstruite
```

Le cas SI montre une autre forme de fonction architecturale :

```text
constante definissante dans un systeme d'unites
composition exacte
realisation pratique
```

Decision :

```text
les fonctions architecturales doivent rester locales a l'architecture qui les rend necessaires.
```

Il ne faut pas transformer toute fonction architecturale locale en famille generale.

| Critere | Saveur-Higgs | Metrologique SI | Effective basse energie | Cosmologique |
|---|---|---|---|---|
| Forme | Constitutive | Definitionnelle | Validite | Inferentielle / reconstructive |
| Corpus | v, Y_f, m_f, CKM, PMNS, Delta m^2 | k_B, N_A, R, c, h, e, K_J, R_K | G_F, M_W, alpha_s, Lambda_QCD, alpha_G, M_Pl | Lambda, H_0, Omega_i, w, A_s, n_s, sigma_8, S_8 |

| Erreur evitee | Famille Saveur-Higgs | Famille Metrologie | Famille Effectif | Famille Cosmologie |

### 14. Regle de reconnaissance d'une architecture

Une architecture doit etre reconnue lorsque les cinq conditions suivantes sont remplies.

```text
1. Plusieurs familles sont necessaires.
2. Une grandeur unique ne porte pas tout le role.
3. Le regime de description impose une solidarite entre fonctions.
4. Le mode d'acces ou de stabilisation est structurant.
5. La creation d'une famille nouvelle serait trop large ou trompeuse.
```

Si ces conditions sont remplies, la bonne decision est :

```text
architecture active de solidarisation
```

et non :

```text
sortie taxonomique trop large
```

### 15. Consequence pour la methode

La methode doit maintenant conserver trois tests distincts.

#### 15.1 Test de famille

Question :

```text
Quelle est la fonction principale de la grandeur ?
```

Reponse attendue :

```text
Couplage, Echelle, Raccordement, Orientation, Convention, ou Relation seulement apres test de non-reduction.
```

#### 15.2 Test de fonction transversale ou architecturale

Question :

```text
Cette grandeur joue-t-elle un role secondaire, transversal ou global ?
```

Reponse possible :

```text
Seuil, Validite, Borne, Hierarchie, Dephasage
ou fonction architecturale locale.
```

#### 15.3 Test d'architecture

Question :

```text
Plusieurs familles deviennent-elles solidaires dans un regime donne ?
```

Si oui :

```text
classer au niveau Architecture active.
```

### 16. Ce que la synthese stabilise

La synthese stabilise dix decisions.

```text
1. Les architectures actives forment un niveau methodologique stable.
2. Une architecture stabilise une solidarisation de fonctions.
3. Les quatre architectures reconnues ont des formes differentes.
4. Saveur-Higgs est une architecture constitutive.
5. Metrologique SI est une architecture definitionnelle.
6. Effective basse energie est une architecture de validite.
7. Cosmologique est une architecture inferentielle et reconstructive.
8. Le regime d'acces participe a la qualification du statut et du classement.
9. La stabilisation doit etre qualifiee.
10. La limite de ce que la grandeur ne dit pas fait partie du classement.
```

### 17. Ce que la synthese ne fait pas

La synthese ne cree pas :

```text
une sortie taxonomique Architecture
une sortie taxonomique Systeme
une sortie taxonomique Metrologie
une sortie taxonomique Effectif
une sortie taxonomique Cosmologie
une sortie taxonomique Saveur-Higgs
```

Elle ne resout pas non plus :

```text
l'origine des constantes
les tensions cosmologiques
la completion quantique de la gravitation
l'origine des hierarchies de saveur
la philosophie complete de la mesure
```

Elle fixe seulement une carte de comparaison entre architectures.

### 20. Formule de cloture

Formule longue :

> Une architecture active apparait lorsqu'un regime de description rend solidaires plusieurs fonctions, roles, operations, acces ou modes de stabilisation, sans que cette solidarite doive devenir une sortie taxonomique superieure.

Formule courte :

> Une architecture est une solidarite de fonctions, non une nouvelle famille.
