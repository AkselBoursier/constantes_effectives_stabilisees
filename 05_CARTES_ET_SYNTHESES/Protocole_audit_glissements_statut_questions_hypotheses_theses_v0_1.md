# Protocole d'audit des glissements de statut — questions, hypothèses, résultats et thèses v0.1

## 0. Statut

```text
statut : protocole actif de la deuxième passe fondationnelle ;
date : 21 juillet 2026 ;
branche : agent/crise-fondatrice-audit-fondations ;
fonction : reconstruire et contrôler les promotions de statut dans le corpus ;
ne vaut pas : obligation de produire une thèse générale, protocole éditorial universel,
               ni remplacement des méthodes scientifiques propres à chaque cycle.
```

## 1. Problème traité

Le corpus contient plusieurs sortes d'énoncés dont les statuts ont parfois migré sans
acte explicite :

```text
intuition ;
question exploratoire ;
question locale ;
question directrice ;
hypothèse de travail ;
interprétation possible ;
résultat physique ou computationnel ;
distinction méthodologique ;
proposition philosophique ;
thèse candidate ;
thèse défendue ;
décision éditoriale.
```

Une phrase peut rester identique tout en changeant de fonction. Le risque ne réside donc
pas seulement dans son contenu, mais dans sa **promotion silencieuse**.

Le protocole cherche à rendre ces migrations visibles et réversibles.

## 2. Principe directeur

```text
aucun énoncé ne monte de rang par répétition, centralité documentaire,
cohérence inter-agents ou simple accumulation de cas.
```

Toute promotion doit préciser :

1. son statut antérieur ;
2. son nouveau statut ;
3. le motif du changement ;
4. les preuves ou arguments nouveaux ;
5. la décision humaine explicite ;
6. les conditions de rétrogradation.

## 3. Registre minimal des statuts

### Q0 — intuition ou motif

```text
fonction : orienter l'attention ;
preuve requise : aucune ;
peut être : physique, philosophique, autobiographique ou heuristique ;
ne vaut pas : question bien posée ni hypothèse.
```

### Q1 — question exploratoire

```text
fonction : ouvrir un paysage et sélectionner des cas ;
preuve requise : pertinence minimale et possibilité d'instruction ;
ne vaut pas : présupposition de sa réponse ni promesse de thèse.
```

### Q2 — question locale opérationnalisée

```text
fonction : organiser un dossier, une recherche bibliographique, un calcul ou une fiche ;
preuve requise : objets, transformations, sources ou opérations identifiables ;
ne vaut pas : généralisation intersectorielle.
```

### H1 — hypothèse de travail

```text
fonction : guider provisoirement une enquête ;
preuve requise : raisons initiales et discriminants ;
doit pouvoir être : confirmée, révisée, abandonnée ou rendue non pertinente.
```

### R1 — résultat local

```text
fonction : établir ce qui a été trouvé dans un périmètre précis ;
preuve requise : source, calcul, argument ou reproduction adaptée au cas ;
ne vaut pas : interprétation philosophique générale.
```

### M1 — distinction ou résultat méthodologique local

```text
fonction : modifier la manière d'instruire un cas ou une famille de cas ;
preuve requise : conséquence observable sur l'enquête ;
ne vaut pas : thèse sur la réalité.
```

### P1 — proposition philosophique locale

```text
fonction : formuler ce qu'un dossier autorise à discuter philosophiquement ;
preuve requise : reconstruction loyale de la physique et confrontation aux voisinages ;
ne vaut pas : originalité ni généralité.
```

### T1 — thèse candidate

```text
fonction : proposition argumentative susceptible de devenir un livrable ;
preuve requise : domaine, objections, antériorités, conséquences et conditions d'échec ;
exige : acte explicite de promotion par l'auteur.
```

### T2 — thèse défendue

```text
fonction : engagement central d'un article, essai ou programme délimité ;
preuve requise : argument complet, état de l'art, cas probants, objections et portée ;
ne devient jamais : centre rétroactif de tous les travaux antérieurs.
```

### E1 — décision éditoriale

```text
fonction : choisir un format, un article, une synthèse ou une publication ;
ne vaut pas : preuve scientifique ou philosophique.
```

## 4. Test généalogique par énoncé

Pour chaque proposition importante, produire une ligne :

| Champ | Question |
|---|---|
| Formulation | quelle phrase ou idée est suivie ? |
| Première trace | où apparaît-elle ? |
| Statut initial | Q0, Q1, Q2, H1, R1, M1, P1, T1, T2 ou E1 ? |
| Question servie | à quel problème répondait-elle alors ? |
| Transformation | comment son sens ou sa fonction a-t-il changé ? |
| Promoteur textuel | auteur, modèle, document de synthèse ou décision distribuée ? |
| Validation humaine | explicite, implicite, absente ou ambiguë ? |
| Gain nouveau | quelle preuve ou conséquence justifiait la promotion ? |
| Risque | surpromotion, sous-qualification, oubli de provenance ou immunisation ? |
| Statut proposé | rang actuel après audit |

Le champ `promoteur textuel` décrit une trace, non une propriété intellectuelle.

## 5. Test de promotion

Une promotion est recevable seulement si les cinq conditions suivantes sont réunies.

### P-A — nouvel apport

Une donnée, un calcul, une comparaison, une objection ou une lecture nouvelle modifie
réellement le dossier.

### P-B — différence de fonction

Le nouveau rang est nommé. Une question n'est pas appelée hypothèse, ni une hypothèse
appelée résultat, seulement parce qu'elle organise plusieurs fichiers.

### P-C — résistance

Le nouvel énoncé possède une condition d'échec adaptée à son rang.

### P-D — voisinage

Avant le passage de P1 à T1, la littérature la plus proche est recherchée. Avant Q2 ou
H1, une recherche exhaustive d'originalité n'est pas requise ; il suffit d'établir que la
question est instruisible et non manifestement mal formée.

### P-E — décision humaine

Le changement de statut est explicitement accepté par l'auteur. La simple validation d'un
document, d'un plan ou d'un workflow ne vaut pas nécessairement adoption de toutes les
promotions qu'il contient.

## 6. Test de rétrogradation

La rétrogradation n'est pas un échec documentaire. Elle peut être la correction normale
d'un projet vivant.

```text
T2 -> T1 : thèse défendue devenue trop large ;
T1 -> P1 : proposition intéressante mais non suffisamment originale ou générale ;
P1 -> M1 : portée philosophique absorbée, gain méthodologique local conservé ;
H1 -> Q2 : hypothèse non soutenue, question encore féconde ;
R1 -> non établi : preuve insuffisante ou résultat non reproduit ;
Q2 -> archive : question devenue sans intérêt ou mal posée.
```

Toute rétrogradation doit conserver les résultats locaux indépendants qui restent valables.

## 7. Séparation des quatre plans de travail

### 7.1 Enquête physique

Objet : mécanismes, modèles, données, transformations, calculs et contraintes.

Sorties légitimes : Q2, H1, R1 et parfois M1.

La question principale est :

> Qu'avons-nous appris physiquement, même si aucune thèse transversale n'en résulte ?

### 7.2 Réflexion philosophique locale

Objet : présupposés, demandes d'explication, portée, voisinages et conséquences conceptuelles
d'un dossier précis.

Sorties légitimes : P1, parfois T1 après promotion explicite.

La question principale est :

> Que cette physique nous conduit-elle à distinguer ou à reposer, et qu'en dit déjà la
> philosophie compétente ?

### 7.3 Méthode et infrastructure

Objet : provenance, reproductibilité, versions, audits, séparation des rangs et réduction
des hallucinations.

Sorties légitimes : M1 et décisions d'intégrité.

La question principale est :

> Le dispositif rend-il le travail contrôlable sans transformer ses propres règles en
> conclusions ?

### 7.4 Édition et publication

Objet : choix d'un livrable, d'une audience et d'un argument.

Sorties légitimes : T1, T2 et E1.

La question principale est :

> Parmi les résultats déjà acquis, existe-t-il maintenant un argument publiable ?

L'édition ne doit pas être introduite comme destination automatique de toute exploration.

## 8. Temporalité révisée de la littérature

Le défaut identifié n'est pas l'absence de vérification, mais le mauvais moment ou la
mauvaise fonction de certaines vérifications.

### Avant un cycle

Recherche légère mais obligatoire :

```text
état physique du problème ;
termes indigènes ;
principales revues ou sources ;
risque que la question repose sur une confusion élémentaire ;
matériaux accessibles.
```

But : poser une question instruisible, non prouver son originalité philosophique.

### Pendant le cycle

Recherche dirigée par les obstacles :

```text
sources primaires ;
méthodes ;
valeurs ;
modèles concurrents ;
limites et controverses techniques.
```

But : répondre correctement à Q2 et tester H1.

### Après un résultat local

Recherche de voisinage :

```text
le résultat était-il déjà connu ?
le dossier produit-il une conséquence nouvelle ?
quelle interprétation la littérature soutient-elle ou interdit-elle ?
```

But : décider entre R1, M1 et P1.

### Avant une thèse ou une publication

Recherche d'originalité et d'antériorité approfondie.

But : décider si P1 peut être promue en T1 ou T2.

## 9. Place du fine-tuning

Le fine-tuning est traité comme une lignée de questions, non comme une conclusion à
retrouver.

Le protocole doit reconstruire au moins :

```text
intuition anthropique ou philosophique initiale ;
questions physiques dans lesquelles elle a été traduite ;
paramètres supposés indépendants ;
dépendances retirées ;
mécanismes testés ;
résultats négatifs ;
dettes déplacées ;
rôle éventuel restant de l'argument anthropique.
```

Chaque résultat reçoit son rang propre. Le cycle 10 ne doit ni disparaître dans la crise
fondatrice ni être utilisé pour sauver une thèse générale.

## 10. Adaptation du test de résistance externe

Le lecteur extérieur n'a plus pour première mission de décider si le projet possède une
thèse transversale originale.

La résistance externe est distribuée selon l'objet :

| Lecteur | Objet principal |
|---|---|
| physicien du secteur | exactitude des dossiers et pertinence des questions locales |
| philosophe du domaine | portée, antériorités et intérêt des propositions P1 ou T1 |
| spécialiste de méthode ou science ouverte | valeur et limites du workflow |
| éditeur ou chercheur expérimenté | possibilité d'un livrable argumentatif borné |

Le dossier court déjà produit reste utile pour tester la thèse transversale historiquement
apparue. Il ne représente plus à lui seul un test de survie du corpus.

## 11. Procédure de deuxième passe

### Étape A — Chronologie des statuts

Reconstituer les transitions suivantes :

```text
question initiale et premiers cycles ;
naissance de la grille et des catégories ;
introduction du pan philosophique ;
passage à la question directrice ;
passage au premier dégagement de thèse ;
quarantaine philosophique ;
cycles 7 à 10 et retour du fine-tuning ;
cycle zéro et corrections successives ;
D1–D5 et reprise QCD ;
crise fondatrice et surcorrection inverse.
```

### Étape B — Tableau des promotions

Pour chaque transition, indiquer :

```text
ce qui a changé de rang ;
si l'auteur l'a explicitement décidé ;
ce que les modèles ont amplifié ;
ce que GitHub a figé ;
ce que les résultats justifiaient réellement.
```

### Étape C — Inventaire des valeurs indépendantes

Pour chaque cycle :

```text
questions encore fécondes ;
résultats physiques ou computationnels ;
résultats négatifs ;
compétences ou infrastructures acquises ;
propositions philosophiques locales ;
éléments devenus triviaux ou déjà connus ;
dettes réelles.
```

### Étape D — Retour au programme ouvert

Décider ensuite seulement :

```text
quelles recherches physiques poursuivre ;
quelles questions abandonner ;
quelles propositions philosophiques laisser locales ;
quels résultats pourraient former un article ;
si une thèse transversale mérite encore d'être testée.
```

## 12. Garde-fou contre les deux surcorrections

```text
surcorrection 1 : une question organisée devient une thèse générale ;
interdiction : promotion sans acte explicite et preuves nouvelles.

surcorrection 2 : l'échec d'une thèse tardive invalide l'enquête qui l'a précédée ;
interdiction : juger les résultats locaux par une destination qui n'était pas leur contrat.
```

## 13. Critère de réussite

Le protocole réussit s'il permet de dire séparément :

```text
ce que le projet cherchait ;
ce qu'il a effectivement trouvé ;
ce qu'il a appris à demander ;
ce qu'il a cru trop tôt pouvoir soutenir ;
ce qui reste philosophiquement intéressant ;
ce qui est seulement documentaire ou pédagogique ;
ce qui mérite une recherche physique supplémentaire ;
ce qui peut être abandonné sans perte.
```

Il échoue s'il produit immédiatement une nouvelle question directrice unique, une nouvelle
thèse de remplacement ou une nouvelle architecture obligatoire.

## 14. Décision de phase

```text
deuxième passe : ouverte ;
objet : généalogie et contrôle des changements de statut ;
thèse générale : non requise ;
recherches physiques locales : conservées ;
verdict final sur le corpus : suspendu ;
prochaine sortie : chronologie raisonnée des glissements et inventaire des valeurs
                   indépendantes par cycle.
```
