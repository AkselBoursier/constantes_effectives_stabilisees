# Seuil de suffisance avant cercle 2 v0.1

## Remontee ciblee des sources avant reprise des architectures

### 1. Fonction

Cette note fixe le seuil de suffisance avant l'ouverture du cercle 2 des
architectures actives.

Elle repond a une hesitation legitime :

```text
faut-il ouvrir le cercle 2 tout de suite,
ou poursuivre toute la remontee des sources DOCX ?
```

Decision :

```text
ne pas ouvrir encore un cercle 2 complet ;
ne pas remonter non plus tout le corpus physique avant de continuer ;
effectuer une remontee ciblee des sources qui conditionnent directement
la solidite des architectures actives.
```

### 2. Raison

Les lots 0, 2 et 3 ont montre que les sources DOCX ne sont pas seulement
genealogiques.

Elles peuvent modifier :

```text
le rang d'une notion ;
le statut d'une ancienne categorie ;
le type d'architecture a reconnaitre ;
la frontiere entre fonction directrice, role contextuel et architecture.
```

Il serait donc imprudent d'ouvrir le cercle 2 sans quelques sources physiques
complementaires.

Mais une remontee exhaustive produirait une masse documentaire trop large
avant le travail architectural.

### 3. Principe retenu

Principe :

```text
remonter ce qui peut modifier une architecture active ;
reporter ce qui ne change pas encore le seuil de decision.
```

Critere de priorite :

```text
une source est prioritaire si elle peut changer :
1. le statut d'une architecture ;
2. la distinction famille / role / architecture ;
3. le regime d'acces ;
4. la trajectoire de stabilisation ;
5. le rattachement d'une grandeur sensible.
```

### 4. Sources a remonter avant cercle 2

Trois blocs sont retenus.

#### 4.1 Bloc thermo-metrologique SI

Sources :

```text
Fiche k_B v0.1.docx
Fiche N_A v0.1.docx
Fiche R v0.1.docx
Synthese du quatrieme cycle.docx
```

Raison :

```text
l'architecture metrologique SI est active,
mais ses sources physiques initiales n'ont pas encore ete remontees.
```

Question a verifier :

```text
l'exactitude conventionnelle absorbe-t-elle la fonction physique,
ou seulement le statut numerique dans un systeme d'unites ?
```

#### 4.2 Bloc Saveur-Higgs cible

Sources :

```text
Fiche v v0.1.docx
Fiche Yukawa v0.1.docx
Fiche CKM v0.1.docx
Fiche Wolfenstein v0.1.docx
Synthese du cycle CKM v0.1.docx
```

Raison :

```text
Saveur-Higgs est deja fortement consolidee,
mais le cercle 2 doit pouvoir verifier la source locale des poles :
echelle, couplages de texture, orientation et coordonnees hierarchiques.
```

Question a verifier :

```text
les relations internes du secteur doivent-elles rester des liaisons qualifiees,
ou etre absorbees par l'architecture constitutive ?
```

#### 4.3 Bloc alpha_G / M_Pl

Sources :

```text
Fiche alpha_G - M_Pl.docx
Fiche alpha_G - M_Pl (1).docx
```

Raison :

```text
le cas alpha_G / M_Pl est deja traite en Markdown,
mais il existe deux DOCX sources distincts.
```

Question a verifier :

```text
les deux versions portent-elles une difference conceptuelle,
ou seulement une variation documentaire locale ?
```

### 5. Sources reportees

Les autres sources physiques restent importantes.

Elles ne sont pas supprimees du programme.

Elles sont reportees parce qu'elles ne conditionnent pas directement
l'ouverture du cercle 2.

Report principal :

```text
lots physiques complets 1A, 1B, 1C ;
genealogie complete des cartes ;
critiques constructives en Markdown source ;
cycle cosmologique source complementaire.
```

### 6. Condition de passage au cercle 2

Le cercle 2 peut etre ouvert lorsque les trois blocs prioritaires ont produit :

```text
1. extractions Markdown de travail ;
2. synthese de recuperation locale ;
3. raccordement a la table de remontee ;
4. verification des anomalies immediate ;
5. synthese de suffisance pre-Cercle 2.
```

Le cercle 2 devra rester un travail architectural.

Il ne devra pas devenir une conversion exhaustive du corpus.

### 7. Formule de cloture

```text
Le seuil de suffisance n'est pas une fermeture des sources ;
c'est la determination de ce qu'il faut relire maintenant
pour que la reprise des architectures ne soit ni prematuree ni diluee.
```
