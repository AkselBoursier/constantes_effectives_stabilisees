# Verdict intermédiaire de la crise fondatrice — première passe v0.1

## 0. Statut

```text
statut : clôture de la première passe interne ;
date : 21 juillet 2026 ;
fonction : indiquer ce que les tests internes autorisent et où commence le blocage externe ;
ne vaut pas : verdict final, décision d’abandon, nouvelle base canonique ou validation
               indépendante de l’audit.
```

## 1. Tests exécutés

| Test | État | Résultat |
|---|---|---|
| soustraction | exécuté | un noyau subsiste, mais seulement sous forme de pouvoir correctif comparatif |
| hypothèse nulle | formulée | concurrente forte, non décorative |
| voisinage | première passe exécutée | la majorité des concepts sont occupés |
| conséquence | exécuté sur cinq cas | aucune conséquence transversale P2 établie |
| discrimination inter-cas | exécuté | seuil de trois cas P2 non atteint |
| échec | défini | réduction et abandon restent autorisés |
| indépendance du workflow | audité | forte intégrité interne, faible résistance externe |
| cible explicative | contrôlée | idée générale déjà occupée ; instrumentation spécialisée non démontrée |

## 2. Thèse qui subsiste

Après soustraction des formulations faibles et des antériorités directes, la seule thèse
candidate encore recevable est :

> Une procédure comparative spécialisée sur les objets appelés constantes peut détecter,
> de manière reproductible, un déplacement non reconnu de la question explicative et en
> tirer une conséquence scientifique ou méthodologique qui n’était pas déjà disponible
> dans les traitements sectoriels.

Cette thèse est beaucoup plus étroite que l’objet initial du programme.

## 3. État des hypothèses

### H-P — contribution comparative

```text
statut : non soutenue à ce stade ;
raison : aucun des cinq cas ne démontre une conséquence transversale P2 ;
charge de la preuve : désormais à H-P.
```

### H-0 — redescription de distinctions existantes

```text
statut : provisoirement favorisée ;
raison : recouvrement fort avec invariance, mesure fondée sur les modèles,
         pragmatisme, perspectivisme, robustesse, EFT/RG, métrologie et explication
         contrastive.
```

### H-D — valeur documentaire

```text
statut : fortement soutenue.
```

### H-M — méthode locale de qualité

```text
statut : soutenue pour la provenance, la reconstruction, la reproductibilité et la
         conservation des limites.
```

### H-L — résultats locaux sans thèse générale

```text
statut : fortement soutenue, notamment pour C2, T1.5 et le cycle 10.
```

## 4. Résultat par cas

```text
QCD / alpha_s : calibration et reproduction locale, pas contribution philosophique propre ;
G_F / EFT : reconstruction correcte d’une grammaire indigène mature ;
SI : distinctions presque entièrement absorbées par la métrologie ;
cosmologie / neutrino : discipline d’inférence utile, originalité non établie ;
cycle 10 : résultat physique local mordant, lien causal avec le cadre transversal non établi.
```

Aucun cas n’est invalidé. Leur fonction dans le programme général est requalifiée.

## 5. Verdict sur le workflow

```text
GitHub : valide l’intégrité des conditions programmées, non leur pertinence ;
LLM : peuvent produire recherche et critique, mais leurs erreurs et cadrages sont corrélés ;
auteur : responsable et réflexif, mais partie prenante du système ;
workflow : fort aux niveaux d’intégrité, reproductibilité interne et reconstruction ;
           insuffisant pour originalité et contribution sans résistance extérieure.
```

La convergence interne n’est plus recevable comme preuve de H-P.

## 6. Ce qui est suspendu

```text
T1.8 : fermé ;
T1.7 : non propagé comme verdict du programme ;
D1–D5 : conservés comme contrôles locaux, non comme preuve de fécondité ;
expansion à de nouveaux cycles : suspendue lorsqu’elle ne répond qu’à la même grille ;
réécriture canonique : suspendue.
```

Les branches et documents existants restent des matériaux, non des autorités automatiques.

## 7. Premier blocage réel

La première passe interne ne peut pas accomplir seule le niveau de résistance externe.
Le prochain test exige au moins un lecteur humain compétent qui ne soit pas engagé dans la
généalogie du corpus.

Le dossier à soumettre ne doit pas être le dépôt complet. Il doit contenir :

```text
une page de thèse H-P ;
une page d’hypothèse H-0 ;
trois cas au maximum ;
les conséquences alléguées ;
les antériorités les plus proches ;
une question explicite : qu’est-ce qui est faux, déjà connu ou réellement ajouté ?
```

## 8. Test externe requis

Le lecteur doit pouvoir choisir librement :

```text
déjà connu ;
clarification pédagogique ;
méthode locale utile ;
conséquence comparative propre ;
erreur ou confusion ;
question sans intérêt suffisant.
```

Le projet échoue si la réponse `conséquence comparative propre` ne peut être soutenue sur
au moins trois cas après discussion des antériorités.

## 9. Option de sauvetage interdite

Il est interdit, avant le test externe, de :

```text
ajouter de nouvelles dimensions à la matrice ;
élargir encore les questions directrices ;
créer une nouvelle ontologie de la constance ;
transformer H-P en simple valeur documentaire ;
compter le cycle 10 comme preuve transversale sans établir le rôle du cadre ;
présenter l’ampleur du corpus comme argument.
```

## 10. Verdict intermédiaire

> La première passe ne détruit pas le projet, mais elle retire la présomption selon
> laquelle son infrastructure et ses distinctions convergent vers une contribution
> philosophique. La valeur actuellement établie est locale, documentaire,
> computationnelle et méthodologique. La thèse transversale est une possibilité étroite,
> non démontrée, et l’hypothèse nulle est provisoirement mieux soutenue.

## 11. Décision de phase

```text
première passe interne : close ;
verdict final : non ouvert ;
PR : non créée ;
fusion : interdite ;
prochaine opération : construire un dossier court pour résistance externe en aveugle.
```
