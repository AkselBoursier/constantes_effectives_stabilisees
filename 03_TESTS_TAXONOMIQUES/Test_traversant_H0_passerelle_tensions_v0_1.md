# Test traversant H_0 - passerelle des tensions v0.1

## Application du lot critique 2 a un cas cosmologique

### 0. Statut documentaire

```text
statut : test traversant ;
cas : H_0 ;
fonction : verifier que la couche decisionnelle produit une sortie active ;
sources :
  02_CYCLES_PHYSIQUES/07_Cycle_cosmologique/Cycle_cosmologique_v0_3_H0.md ;
  02_CYCLES_PHYSIQUES/07_Cycle_cosmologique/Synthese_cycle_cosmologique_v0_1.md ;
  01_CADRE_METHODOLOGIQUE/Addendum_matrice_criblage_lot2_v0_1.md ;
  01_CADRE_METHODOLOGIQUE/Matrice_temporelle_v0_2.md ;
  01_CADRE_METHODOLOGIQUE/Passerelle_escalade_tensions_cosmologiques_v0_1.md.
```

Le test ne cherche pas a recalculer la tension `H_0`.

Il cherche a verifier la decision methodologique.

Question :

```text
la tension H_0 peut-elle devenir un signal methodologique actif
sans devenir une famille, une anomalie conclue ou une these physique ?
```

### 1. Diagnostic de depart

La fiche cosmologique classe deja `H_0` comme :

```text
parametre d'etat actuel.
```

Elle refuse trois sorties :

```text
H_0 comme constante fondamentale ;
tension H_0 comme famille taxonomique ;
mesure locale et inference CMB comme meme objet epistemique.
```

Le test traversant conserve cette base.

Il ajoute la couche decisionnelle :

```text
quel niveau d'escalade la tension H_0 autorise-t-elle ?
```

### 2. Matrice temporelle v0.2

#### Axe A - Constitution physique

| Champ | Application a H_0 |
|---|---|
| Regime constituant | Cosmologie homogene et isotrope, expansion decrite par `a(t)` |
| Condition d'installation | Evaluation actuelle du taux d'expansion : `H_0 = H(t_0)` |
| Dependances physiques | Modele d'expansion, contenu cosmologique, `Lambda`, `Omega_i`, courbure selon cadre |
| Domaine de persistance | Description cosmologique actuelle et comparaisons d'expansion tardive |
| Variation interne | `H(t)` varie ; `H_0` est une valeur d'etat actuelle |
| Test de degradation | Modifier le modele de fond, `r_d`, `Omega_m` ou la calibration modifie l'inference |
| Limite de constitution | Ne donne pas seul la cause de l'expansion ni la nature de l'energie noire |

#### Axe B - Acces epistemique

| Champ | Application a H_0 |
|---|---|
| Routes d'acces | Distance ladder, CMB + LambdaCDM, BAO + calibration, sondes combinees |
| Modele d'inference | Fort pour CMB + LambdaCDM ; explicite pour BAO ; localement dependant des calibrations |
| Degenerescences | `Omega_m`, `r_d`, calibrations de distances, choix de modele de fond |
| Robustesse inter-acces | Routes robustes localement, mais discordantes entre familles d'acces |
| Tension eventuelle | Tension inter-acces persistante |
| Test de substitution | Changer calibration, route locale, ancre BAO ou modele de fond |
| Limite d'acces | Une route ne suffit pas a definir le statut physique global |

Couplage constitution / acces :

```text
fort et conflictuel.
```

Decision temporelle :

```text
H_0 est physiquement un parametre d'etat actuel ;
sa tension appartient au couplage entre routes d'acces, modele et
calibration.
```

### 3. Passerelle d'escalade

Niveau retenu :

```text
niveau 3 : tension persistante.
```

Justification :

| Controle | Lecture |
|---|---|
| Routes d'acces | Plusieurs routes donnent des contraintes precises mais non equivalentes |
| Independance | Les routes ne sont pas toutes independantes ; elles partagent parfois modele, calibration ou ancre |
| Degenerescences | `r_d`, `Omega_m`, `Lambda`, courbure et calibration peuvent redistribuer l'ecart |
| Systematiques | Les systematiques locales et cosmologiques restent centrales |
| Modele | Le contraste CMB/local depend fortement du modele cosmologique |
| Temporalite | La tension porte surtout sur l'acces epistemique, non sur une variation physique directe de `H_0` |
| Substitution | Des substitutions controlees existent, donc le niveau 4 peut etre teste mais non affirme ici |

Sortie :

```text
tension persistante sous stress test ;
test de substitution ouvert ;
pas anomalie candidate conclue.
```

### 4. Test de degradation

Question :

```text
que devient la tension si l'on relache progressivement un noeud du reseau ?
```

| Noeud relache | Effet attendu pour le test |
|---|---|
| Calibration locale | Verifie si la route locale haute depend d'un indicateur dominant |
| Ancre BAO / `r_d` | Teste la sensibilite de l'inference tardive a l'echelle standard |
| Modele LambdaCDM plat | Teste le poids du modele dans l'inference CMB |
| `Omega_m` ou courbure | Teste les degenerescences de fond |
| Combinaison de sondes | Mesure si la tension se concentre ou se distribue |

Diagnostic :

```text
la degradation attendue est distribuee.
```

Le test ne pointe pas un unique noeud deja suffisant.

Il confirme que `H_0` doit rester dans un reseau d'acces.

### 5. Test de substitution

Substitutions controlees a ouvrir :

| Substitution | Question |
|---|---|
| Route locale alternative | Les indicateurs TRGB, JAGB, Cepheides et SNe convergent-ils ? |
| Calibration BAO differente | L'ancrage choisi deplace-t-il la contrainte sur `H_0` ? |
| Modele de fond etendu | Une extension phenomenologique reduit-elle la tension sans cout excessif ? |
| Combinaison multi-sondes | Les contraintes convergent-elles ou revelent-elles une incompatibilite plus large ? |

Decision :

```text
le corpus peut ouvrir des tests de substitution ;
il ne doit pas convertir cette ouverture en verdict physique.
```

### 6. Sortie taxonomique

Sortie pour `H_0` :

```text
classement confirme avec role contextuel explicite.
```

Formulation :

```text
H_0 reste un parametre d'etat actuel du sous-reseau fond / expansion.
La tension H_0 devient une tension persistante sous stress test,
avec test de substitution ouvert.
```

Verrous d'options :

```text
ne pas classer H_0 comme constante fondamentale ;
ne pas classer la tension comme famille ;
ne pas conclure a une anomalie sans dossier autonome ;
ne pas confondre valeur actuelle, mesure locale et inference CMB.
```

### 7. Resultat du test traversant

Le dispositif lot 2 produit bien plus qu'une reserve.

Il produit :

```text
un niveau d'escalade ;
un test de degradation ;
un test de substitution ;
une sortie taxonomique maintenue ;
des verrous d'options.
```

Decision generale :

```text
le cas H_0 valide l'addendum decisionnel, la matrice temporelle v0.2
et la passerelle d'escalade comme outils actifs.
```

### 8. Suite logique

Avant de modifier la carte consolidee, un second test reste utile si le corpus veut verifier la symetrie du dispositif :

```text
S_8 pour la croissance / reconstruction ;
ou masse absolue des neutrinos pour les bornes non equivalentes.
```

Mais le premier test suffit a etablir que le chantier critique peut etre raccorde sans refonte globale immediate.

### 9. Formule de cloture

```text
H_0 n'est pas rendu plus fondamental par sa tension.
Sa tension rend seulement visible le travail du reseau d'acces :
elle oblige a dire quel modele, quelle calibration, quelle route et
quelle substitution portent la decision.
```
