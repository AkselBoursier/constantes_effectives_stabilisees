Cette note accompagne l'ouverture du lot 0 de remontee DOCX.

```text
ce que les sources methodologiques fondatrices confirment ;
ce que la methode active a correctement transforme ;
ce qui ne doit pas etre reactive tel quel ;
et ce qui doit guider les tests taxonomiques puis les architectures.
```

### 3. Resultat general de lecture

Le lot 0 confirme que la methode active ne rompt pas avec le socle.

Elle en deplace l'ordre.

Les sources initiales contiennent deja les noyaux suivants :

```text
constance comme robustesse situee ;
regime physique comme condition de sens ;
acces epistemique comme condition de connaissance ;
fonction comme lecture principale de la grandeur ;
stabilisation comme statut graduel ;
compression taxonomique comme exigence de methode ;
prudence ontologique comme garde-fou.
```

La v1.3 ne doit donc pas etre lue comme une correction exterieure.

Elle est une reprise plus robuste du socle :

```text
la famille n'est plus la premiere question ;
la forme logique vient avant la fonction ;
la fonction directrice vient avant la sortie taxonomique ;
le role contextuel evite de creer de fausses familles ;
le regime d'acces devient aussi important que le regime physique ;
la temporalite explicite ce que les sources appelaient deja regimes, transitions et stabilisations.
```

### 4. Apports par source

#### 4.1 PROJECT_BRIEF

Le brief fixe trois elements encore actifs.

```text
1. Le projet n'est pas une nouvelle theorie physique.
2. Il produit une cartographie methodologique des statuts de constance.
3. Une constante effective stabilisee est une fonction robuste dans un regime.
```

Il contient deja plusieurs precautions qui restent valables :

```text
ne pas reintroduire Seuil comme famille ;
ne pas reintroduire Fond comme famille ;
toujours qualifier les liaisons ;
ne pas confondre famille et architecture ;
ne pas confondre regime physique et regime d'acces.
```

Point a recuperer :

```text
le brief donne une genealogie utile des phases du projet.
Cette genealogie doit servir aux tests et aux architectures,
mais elle ne doit pas dicter le classement actif.
```

#### 4.2 Methode v0.1

La methode v0.1 etablit le protocole de base :

```text
identifier la grandeur ;
distinguer dimension, regime, fonction, acces, dependance d'echelle ;
evaluer la stabilisation ;
noter les limites et les questions ouvertes.
```

Ce protocole reste actif dans la v1.3.

Transformation necessaire :

```text
la categorie proposee en fin d'analyse ne doit plus etre traitee comme la premiere question.
Elle devient une sortie controlee apres forme logique, fonction directrice,
role contextuel, regime d'acces et trajectoire.
```

#### 4.3 Regimes de constance

Cette source est courte mais structurante.

Elle formule l'hypothese de depart :

```text
la constance peut etre une propriete de regime.
```

Elle justifie directement trois couches de la methode actuelle :

```text
regime physique ;
temporalite de constitution ;
conditions de stabilisation.
```

Point a recuperer :

```text
la question "a partir de quand une grandeur devient-elle definissable ?"
doit rester disponible dans les fiches physiques,
surtout pour electrofaible, QCD, neutrinos et cosmologie.
```

#### 4.4 Note de synthese methodologique

La note de synthese methodologique consolide la premiere architecture du cadre.

Elle conserve une valeur importante :

```text
elle rend explicite que les constantes effectives stabilisees
ne sont pas une classe concurrente des constantes physiques ordinaires,
mais une maniere de lire leurs conditions de stabilite, d'acces et de fonction.
```

Point a transformer :

```text
la grille en familles fonctionnelles doit etre relue comme une couche historique.
Elle reste utile pour comprendre la compression,
mais la v1.3 impose des rangs prealables.
```

#### 4.5 Addendum methodologique v0.2

L'addendum apporte trois corrections decisives.

```text
1. Une taxonomie doit compresser.
2. Le regime de definition physique et le regime d'acces epistemique doivent etre distingues.
3. La stabilisation empirique et la stabilisation conventionnelle ne relevent pas du meme axe.
```

Ces trois points sont repris et renforces par la v1.3.

Point a recuperer :

```text
la distinction entre constante definissante,
constante exacte par composition,
et grandeur mesuree dans un systeme
doit rester centrale pour l'architecture metrologique SI.
```

#### 4.6 Note critique de compression

Cette note est le document le plus important pour eviter la proliferation.

Principe encore actif :

```text
une distinction locale peut etre utile dans une fiche,
mais inutile comme famille generale.
```

Elle anticipe plusieurs decisions recentes :

```text
ne pas multiplier les categories locales ;
retrograder les sous-types ;
traiter Seuil comme fonction locale ou transversale ;
traiter Fond comme fonction architecturale ;
surveiller Relation comme famille faible.
```

Point a recuperer :

```text
la compression ne doit pas appauvrir la carte ;
elle doit changer le niveau des distinctions.
```

### 6. Ce qui ne doit pas etre reactive

Le lot 0 ne doit pas servir a restaurer les anciennes formulations.

Ne pas reactiver :

```text
Famille comme premier rang de lecture ;
Relation comme categorie par defaut ;
Seuil comme famille superieure ;
Fond comme famille superieure ;
une categorie nouvelle pour chaque constante difficile ;
une stabilisation unique qui melange mesure, borne, reconstruction et convention ;
une lecture ontologique forte avant la discipline de description.
```

Ces formulations restent historiquement utiles.

Elles ne sont plus des sorties actives sans requalification.

### 7. Recuperations positives

Formulations a conserver sous forme positive :

```text
constance = robustesse fonctionnelle situee ;
la methode classe des modes de stabilisation, non des essences ;
une grandeur est lue dans un regime, par un acces, avec une fonction ;
une taxonomie reussit quand elle compresse sans effacer les differences utiles ;
une architecture decrit une solidarite de fonctions, pas une famille plus grande ;
une tension d'acces peut devenir un test de validite, pas une categorie de plus.
```

### 8. Consequence pour les tests taxonomiques

Le lot 0 impose l'ordre suivant.

```text
1. Ouvrir les tests taxonomiques.
2. Extraire les problemes poses par Seuil, Fond, Relation et familles fragiles.
3. Recuperer les distinctions utiles sans restaurer les anciennes familles.
4. Transformer les anciennes objections en tests de passage.
5. Revenir ensuite aux architectures actives.
```

Les tests taxonomiques ne doivent pas etre lus comme un retour en arriere.

Ils deviennent le banc d'essai de la v1.3.

### 9. Consequence pour les architectures

Avant le cercle 2 des architectures, il faudra verifier :

```text
si chaque architecture est bien une solidarite de fonctions ;
si elle ne masque pas une ancienne famille elargie ;
si ses tensions relevent du regime physique, de l'acces, du modele ou de la temporalite ;
si les sources DOCX d'architecture confirment ou corrigent les notes Markdown actives.
```

La remontee du lot 0 confirme donc l'ordre prevu :

```text
lot 0 methodologique -> tests taxonomiques -> sources d'architecture -> cercle 2.
```

### 10. Formule de cloture

```text
Le socle methodologique ne demande pas de revenir aux anciennes cases ;
il demande de rendre explicite pourquoi ces cases ont ete transformees en rangs, roles, acces, trajectoires et architectures.
```
