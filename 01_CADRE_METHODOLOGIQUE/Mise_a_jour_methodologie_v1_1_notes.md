# Mise a jour methodologique v1.1 - notes preparatoires

Statut :

```text
transformee en Note_synthese_methodologique_v1_1.md
```

## Pourquoi cette mise a jour est necessaire

La note methodologique actuelle est anterieure a plusieurs decisions importantes de la carte.

Elle ne reflete plus completement :

```text
la retrogradation de Seuil
la retrogradation de Fond
la confirmation de Relation sous contrainte
la distinction stabilisee entre famille, fonction transversale, fonction architecturale et architecture
la confirmation de l'architecture effective basse energie
l'ouverture prudente du cycle cosmologique
```

La prochaine note methodologique devra donc etre une version :

```text
Note_synthese_methodologique_v1_1.md
```

et non une simple correction locale de l'ancien texte.

### 1. Decisions a integrer

#### 1.1 Noyau des familles

Le noyau actuel contient six familles superieures :

```text
Couplage
Echelle
Relation
Raccordement
Orientation
Convention
```

La note v1.1 devra supprimer toute presentation de :

```text
Seuil
Fond
```

comme familles superieures.

#### 1.2 Relation sous contrainte

Relation reste une famille, mais seulement si elle est qualifiee.

Sous-types a integrer :

```text
relation stratifiee
relation compositionnelle
relation spectrale
relation constitutive de regime
relation normalisee
relation d'etat
```

Regle a inscrire :

```text
Relation seule est insuffisante.
```

#### 1.3 Fonctions transversales

La note v1.1 devra stabiliser :

```text
Seuil
Borne
Validite
Dephasage
Hierarchie
```

comme fonctions transversales.

Point nouveau fort :

```text
Validite
```

doit etre renforcee apres le cycle effectif basse energie.

#### 1.4 Fonctions architecturales

La note v1.1 devra stabiliser :

```text
Fond
Parametre d'etat
Densite normalisee
Relation d'etat
Condition initiale
```

comme fonctions architecturales.

Point important :

```text
Fond
```

ne doit pas redevenir une famille, meme dans le cycle cosmologique.

### 2. Architectures a integrer

La methode doit maintenant reconnaitre explicitement quatre architectures.

```text
Saveur-Higgs
Metrologique SI
Effective basse energie
Cosmologique
```

Statuts actuels :

| Architecture | Statut |
|---|---|
| Saveur-Higgs | Confirmee comme architecture inter-familles |
| Metrologique SI | Confirmee, avec extension a tester |
| Effective basse energie | Confirmee par le cycle G_F / M_W / alpha_s / Lambda_QCD / alpha_G / M_Pl |
| Cosmologique | Necessaire, ouverte prudemment |

La note v1.1 devra insister sur la regle :

```text
Une architecture n'ajoute pas une famille.
Elle montre comment plusieurs familles deviennent solidaires dans un regime.
```

### 3. Mise a jour de la definition generale

Definition actuelle a conserver :

> Une constante effective stabilisee n'est pas seulement une valeur stable ; c'est une fonction robuste dans un regime, connue par un acces determine, stabilisee selon un mode precis, et parfois inscrite dans une architecture plus vaste.

Definition v1.1 a enrichir :

```text
Une constante effective stabilisee doit etre analysee selon :
1. sa famille principale ;
2. ses fonctions transversales ;
3. ses fonctions architecturales eventuelles ;
4. son regime de definition physique ;
5. son regime d'acces epistemique ;
6. son mode de stabilisation ;
7. son appartenance eventuelle a une architecture inter-familles.
```

### 4. Mise a jour de la matrice canonique

La matrice v1.1 devra reprendre la matrice v0.9 :

| Critere | Question |

| Grandeur | De quoi parle-t-on exactement ? |
| Type logique | Valeur, rapport, couplage, echelle, angle, phase, matrice, coefficient, parametre ? |
| Dimension | Dimensionnee ou sans dimension ? |
| Famille principale | Couplage, Echelle, Relation, Raccordement, Orientation ou Convention ? |
| Sous-type local | Quelle qualification precise est necessaire ? |
| Fonction transversale eventuelle | Seuil, borne, validite, dephasage, hierarchie ? |

### 5. Corrections a apporter a l'ancienne note

L'ancienne note methodologique doit etre corrigee sur plusieurs points.

#### 5.1 Retirer Fond et Seuil des familles

Si l'ancienne note presente encore :

```text
Fond
Seuil
```

comme familles, la v1.1 devra les replacer.

```text
Seuil -> fonction transversale
Fond  -> fonction architecturale
```

#### 5.2 Integrer la distinction famille / architecture

La v1.1 devra avoir une section autonome :

```text
Famille, fonction, architecture
```

avec les definitions :

```text
famille = fonction principale d'une grandeur
fonction transversale = role secondaire ou transversal
fonction architecturale = role dans une configuration globale
architecture = articulation de plusieurs familles et fonctions
```

#### 5.3 Integrer l'architecture effective basse energie

La v1.1 devra integrer la formule :

```text
Architecture effective basse energie =
Raccordement
+ Echelle de validite
+ Couplage de regime
+ Fonctions Validite / Seuil / Hierarchie
```

et la decision :

```text
Effectif ne devient pas une famille.
```

#### 5.4 Integrer la prudence cosmologique

La v1.1 devra ajouter une section :

```text
Cas cosmologiques : prudence de rang
```

avec la regle :

```text
ne pas confondre constante, parametre de modele, observable reconstruite, condition initiale et tension de donnees.
```
