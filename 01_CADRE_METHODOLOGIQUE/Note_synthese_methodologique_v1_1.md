# Note de synthese methodologique v1.1

## Constantes effectives stabilisees : familles, fonctions, regimes, acces et architectures

### 1. Fonction de la v1.1

Cette note remplace la lecture methodologique v1.0 sur les points ou elle est devenue trop large.

Elle integre les decisions issues de :

```text
tests de Seuil, Fond et Relation
consolidation saveur-Higgs
cycle effectif basse energie
carte consolidee v0.9
cycle cosmologique Lambda / H_0 / Omega_i / w / A_s / n_s / sigma_8 / S_8
architecture cosmologique
extension metrologique SI c / h / e
carte consolidee v1.1
synthese des architectures inter-familles
```

La correction principale est une correction de rang :

```text
Seuil n'est plus une famille.
Fond n'est plus une famille.
Effectif n'est pas une famille.
Cosmologie n'est pas une famille.
```

La v1.1 stabilise donc une methode a quatre niveaux :

```text
famille fonctionnelle
fonction transversale
fonction architecturale
architecture inter-familles
```

### 2. Definition directrice

Definition conservee :

> Une constante effective stabilisee n'est pas seulement une valeur stable ; c'est une fonction robuste dans un regime, connue par un acces determine, stabilisee selon un mode precis, et parfois inscrite dans une architecture plus vaste.

La v1.1 precise cette definition.

Une grandeur doit etre analysee selon :

```text
1. sa famille principale ;
2. ses fonctions transversales eventuelles ;
3. ses fonctions architecturales eventuelles ;
4. son regime de definition physique ;
5. son regime d'acces epistemique ;
6. son mode de stabilisation ;
7. son appartenance eventuelle a une architecture inter-familles ;
8. ses limites de validite et d'interpretation.
```

Formule courte :

> Une constante physique est moins un nombre isole qu'un point de stabilite dans une architecture de regimes, d'acces et de fonctions.

### 3. Difference entre v1.0 et v1.1

La v1.0 avait encore tendance a traiter comme familles certaines categories utiles mais trop larges.

La v1.1 corrige cela.

| Element | Statut v1.0 | Statut v1.1 |
|---|---|---|
| Seuil | Famille fragile ou provisoire | Fonction transversale |
| Fond | Famille fragile ou candidate | Fonction architecturale |
| Relation | Famille large | Famille conservee sous qualification obligatoire |
| Effectif basse energie | Architecture en test | Architecture inter-familles confirmee |
| Validite | Fonction secondaire implicite | Fonction transversale centrale |
| Cosmologie | Architecture candidate | Architecture inter-familles confirmee |
| Tensions de donnees | Probleme d'interpretation | Propriete du regime d'acces |

Regle :

```text
Une categorie peut etre indispensable sans etre une famille superieure.
```

### 4. Noyau des familles superieures

La v1.1 retient six familles fonctionnelles superieures :

```text
Couplage
Echelle
Relation
Raccordement
Orientation
Convention
```

Ces familles repondent a la question :

```text
que fait principalement cette grandeur ?
```

Elles ne repondent pas encore a la question :

```text
dans quelle architecture cette grandeur prend-elle place ?
```

Cette seconde question releve du niveau architectural.

### 5. Famille Couplage

Une constante de couplage parametre une intensite d'interaction ou de liaison entre champs, particules ou secteurs.

Exemples :

```text
alpha
alpha_s(Q^2)
alpha_G(E)
y_f
```

Sous-types utiles :

```text
couplage de regime
couplage courant
couplage de texture
couplage effectif construit par rapport a une echelle
```

Regle :

```text
Un couplage dimensionne doit etre lu par ses effets, ses rapports ou sa reformulation sans dimension.
```

Exemple :

```text
G_N seul n'est pas comparable directement a alpha.
alpha_G(E) ~ (E / M_Pl)^2 rend lisible le couplage gravitationnel effectif.
```

### 6. Famille Echelle

Une constante d'echelle fixe, revele ou hierarchise un ordre de grandeur structurant pour un regime physique.

Exemples :

```text
v
M_W
Lambda_QCD
M_Pl
G_F^(-1/2)
```

Sous-types utiles :

```text
echelle structurante
echelle electrofaible de jauge
echelle dynamique
echelle de validite attendue
echelle derivee
```

Regle :

```text
Une echelle peut jouer une fonction seuil ou validite sans que Seuil devienne famille.
```

Exemples :

```text
M_W        : Echelle, avec fonction de validite pour la theorie de Fermi
Lambda_QCD : Echelle dynamique, avec fonction seuil perturbatif / non perturbatif
M_Pl       : Echelle de validite attendue, avec fonction hierarchique
```

### 7. Famille Relation

Une constante relationnelle stabilise une liaison entre grandeurs, regimes, niveaux, spectres ou composantes.

La famille Relation est conservee, mais elle est sous contrainte.

Regle stricte :

```text
Ne jamais ecrire seulement "famille Relation".
Toujours preciser relation de quoi, entre quoi, dans quel regime, selon quel acces.
```

Sous-types stabilises :

```text
relation stratifiee
relation compositionnelle
relation spectrale
relation constitutive de regime
relation normalisee
relation d'etat
```

Exemples :

```text
m_p / m_e             : relation stratifiee
R = N_A k_B           : relation compositionnelle
Delta m^2             : relation spectrale
m_f = y_f v / sqrt(2) : relation constitutive de regime
Omega_i               : relation normalisee
w = p / rho           : relation d'etat
```

Relation reste une famille parce qu'elle designe une fonction propre :

```text
stabiliser une liaison
```

Mais elle reste faible si elle n'est pas sous-typee.

### 8. Famille Raccordement

Une constante de raccordement relie une description effective a une description plus complete, ou deux niveaux theoriques distincts.

Exemple central :

```text
G_F
```

Lecture :

```text
G_F raccorde l'interaction faible basse energie a la theorie electrofaible avec boson W explicite.
```

Sous-types utiles :

```text
raccordement effectif basse energie
raccordement local
raccordement architectural
raccordement attendu
```

Regle :

```text
Tout raccordement implique des relations, mais toute relation n'est pas un raccordement.
```

Exemples :

```text
G_F : raccordement local
alpha_s(Q^2) / Lambda_QCD : raccordement architectural de regime
alpha_G(E) / M_Pl : raccordement attendu vers une completion ouverte
```

### 9. Famille Orientation

Une constante d'orientation stabilise un desalignement entre bases, champs, etats ou secteurs d'interaction.

Exemples :

```text
theta_W
CKM
PMNS
delta_CP
parametres de Wolfenstein
```

Sous-types utiles :

```text
orientation de jauge
orientation de saveur transitionnelle
orientation de saveur oscillatoire
orientation complexe CP
coordonnees hierarchiques d'orientation
```

Regles :

```text
CKM oriente dans le secteur quark.
PMNS oriente dans le secteur leptonique.
Delta m^2 ne doit pas etre classe comme orientation : il dephase.
Wolfenstein ne cree pas une famille : il coordonne localement une orientation.
```

### 10. Famille Convention

Une constante conventionnelle recoit une valeur exacte par definition dans un systeme d'unites, ou devient exacte par composition de constantes definissantes.

Exemples :

```text
k_B
N_A
R = N_A k_B
```

Sous-types :

```text
constante definissante primaire
constante exacte par composition
```

Regle :

```text
Exactitude conventionnelle n'est pas precision empirique.
```

Une valeur peut etre exacte dans le SI sans que son role physique soit epuise par la convention.

### 11. Fonctions transversales

Une fonction transversale n'est pas une famille superieure.

Elle designe un role secondaire, lateral ou recurrent qu'une grandeur peut jouer en plus de sa fonction principale.

Fonctions transversales stabilisees :

```text
Seuil
Borne
Validite
Dephasage
Hierarchie
```

#### 11.1 Seuil

Une fonction seuil marque une limite, un passage, un plancher ou un ancrage.

Regle :

```text
Seuil = fonction transversale, non famille.
```

Formulation :

> Un seuil est souvent la maniere dont une echelle devient visible depuis une description qui cesse d'etre suffisante.

#### 11.2 Borne

Une fonction borne designe un acces par limite experimentale ou observationnelle sans mesure positive complete.

Exemples :

```text
masse absolue des neutrinos
contraintes cosmologiques modele-dependantes
limites experimentales directes
```

La borne appartient souvent davantage au regime d'acces qu'a la fonction physique principale.

#### 11.3 Validite

Validite devient centrale en v1.1.

Elle repond a la question :

```text
quelle description vaut dans quel domaine ?
```

Elle est plus precise que Seuil.

```text
Seuil   : ou se situe une limite ou zone de passage ?
Validite : quelle description reste applicable dans quel domaine ?
```

Exemples :

```text
M_W        : validite de la theorie de Fermi
Lambda_QCD : validite relative des descriptions perturbatives en QCD
M_Pl       : validite attendue de la gravitation effective basse energie
```

#### 11.4 Dephasage

Une fonction de dephasage decrit la manifestation dynamique d'une relation spectrale sous forme de phase.

Exemple :

```text
Delta m^2
```

Regle :

```text
Delta m^2 dephase.
PMNS oriente.
```

#### 11.5 Hierarchie

Une fonction hierarchique organise des ecarts de taille, de masse, d'echelle ou de melange.

Exemples :

```text
parametres de Wolfenstein
hierarchies de Yukawa
E << M_W
Q par rapport a Lambda_QCD
E << M_Pl
```

Regle :

```text
Hierarchie = fonction transversale forte, non famille superieure.
```

### 12. Fonctions architecturales

Une fonction architecturale prend sens dans une configuration globale.

Elle ne forme pas une famille superieure.

Fonctions architecturales stabilisees :

```text
Fond
Parametre d'etat
Densite normalisee
Relation d'etat
Condition initiale
Observable reconstruite
```

#### 12.1 Fond

Fond designe un role de structure globale, surtout dans l'architecture cosmologique.

Exemple :

```text
Lambda
```

Regle :

```text
Fond n'est pas une famille superieure.
```

Formulation stabilisee :

> Lambda est un terme de fond dans l'architecture cosmologique ; sa valeur est fortement stabilisee comme parametre du modele LambdaCDM, mais son statut physique profond et son caractere strictement constant restent ouverts.

#### 12.2 Parametre d'etat

Un parametre d'etat decrit une coordonnee actuelle ou globale d'un systeme cosmologique.

Exemple :

```text
H_0
```

Regle :

```text
H_0 est une valeur actuelle du taux d'expansion, non une constante fondamentale.
```

#### 12.3 Densite normalisee

Une densite normalisee rapporte une composante a une densite de reference.

Exemples :

```text
Omega_m
Omega_Lambda
Omega_b
Omega_c
```

Statut attendu :

```text
Relation normalisee + fonction architecturale Densite normalisee
```

#### 12.4 Relation d'etat

Une relation d'etat decrit le lien entre pression et densite d'une composante.

Exemple :

```text
w = p / rho
```

Regle :

```text
w n'est pas Lambda.
```

Pour une composante Lambda stricte :

```text
w = -1
```

Mais w peut aussi servir de parametrisation phenomenologique dans des modeles d'energie noire dynamique.

#### 12.5 Condition initiale

Une condition initiale stabilise un parametre d'entree du modele cosmologique, souvent lie aux perturbations primordiales.

Exemples attendus :

```text
A_s
n_s
```

Ces grandeurs ne doivent pas etre rangees dans Fond par simple association cosmologique.

#### 12.6 Observable reconstruite

Une observable reconstruite condense une grandeur cosmologique derivee d'un reseau d'observations, de modeles et de corrections systematiques.

Exemples :

```text
sigma_8
S_8
```

Regle :

```text
une observable reconstruite n'est pas une constante primitive.
```

### 13. Architectures inter-familles

Une architecture inter-familles articule plusieurs familles et fonctions dans un secteur, un regime ou un systeme de description.

Une famille repond a :

```text
que fait principalement cette grandeur ?
```

Une architecture repond a :

```text
comment plusieurs fonctions deviennent-elles solidaires dans un regime ?
```

Regle centrale :

```text
Une architecture n'ajoute pas une famille.
Elle montre comment plusieurs familles deviennent solidaires dans un regime de description.
```

Architectures reconnues :

```text
Saveur-Higgs
Metrologique SI
Effective basse energie
Cosmologique
```

### 14. Architecture saveur-Higgs

L'architecture saveur-Higgs articule :

```text
Echelle
+ Couplage
+ Relation
+ Orientation
```

Lecture :

```text
v          : echelle electrofaible
Y_f        : couplages de texture
m_f        : masses stabilisees par relation constitutive de regime
CKM        : orientation de saveur transitionnelle
PMNS       : orientation de saveur oscillatoire
Delta m^2  : relations spectrales dans le secteur neutrino
```

Regle :

```text
Saveur-Higgs n'est pas une famille.
```

Elle evite de fragmenter le secteur en categories locales trop nombreuses.

### 15. Architecture metrologique SI

L'architecture metrologique SI articule :

```text
Convention
+ Relation
+ Echelle
```

Exemples :

```text
k_B : constante definissante primaire du kelvin
N_A : constante definissante primaire de la mole
R = N_A k_B : constante exacte par composition
c : constante definissante primaire, relation espace-temps / echelle causale
h : constante definissante primaire, quantum d'action
e : constante definissante primaire, quantum de charge
K_J = 2 e / h : constante exacte par composition
R_K = h / e^2 : constante exacte par composition
```

Regle :

```text
Une constante definissante peut rester physiquement operatoire.
Son exactitude vient du systeme de definition, non d'une mesure plus precise.
```

Note d'integration produite :

```text
04_ARCHITECTURES/Architecture_metrologique_SI_notes.md
```

### 16. Architecture effective basse energie

L'architecture effective basse energie est confirmee.

Elle articule :

```text
Raccordement
+ Echelle de validite
+ Couplage de regime
+ Fonctions Validite / Seuil / Hierarchie
```

Corpus stabilise :

```text
G_F
M_W
alpha_s(Q^2)
Lambda_QCD
alpha_G(E)
M_Pl
```

Trois formes internes d'effectivite sont distinguees :

| Forme | Cas | Structure |
|---|---|---|
| Integration d'un mediateur massif | G_F / M_W | Un degre de liberte lourd n'est pas resolu ; son effet devient coefficient local |
| Flot de couplage et changement de regime | alpha_s(Q^2) / Lambda_QCD | Le couplage varie avec l'echelle ; une echelle dynamique marque le changement de description |
| Couplage dimensionne et suppression d'echelle | alpha_G(E) / M_Pl | Le couplage effectif depend d'un rapport d'echelle ; la basse energie est un domaine de suppression |

Regles :

```text
Effectif ne devient pas une famille.
Raccordement peut etre local, distribue ou attendu.
Validite devient une fonction transversale centrale.
```

Formule :

> L'effectivite stabilise moins une valeur qu'une validite de description.

### 17. Architecture cosmologique

L'architecture cosmologique est confirmee, mais delicate.

Elle articule des objets de types differents :

```text
Lambda
H_0
Omega_m, Omega_Lambda
w
A_s, n_s
sigma_8, S_8
```

Statuts de travail :

| Objet | Statut methodologique |
|---|---|
| Lambda | Terme de fond |
| H_0 | Parametre d'etat actuel |
| Omega_i | Densite normalisee / relation normalisee |
| w | Relation d'etat |
| A_s, n_s | Conditions initiales perturbatives |
| sigma_8, S_8 | Observables ou amplitudes reconstruites de structure |

Regles de prudence :

```text
ne pas transformer cosmologie en famille ;
ne pas reintroduire Fond comme famille ;
ne pas confondre constante, parametre de modele, observable reconstruite, condition initiale et tension de donnees ;
ne pas transformer une tension en conclusion physique directe.
```

### 18. Regime physique et regime d'acces

La distinction est obligatoire.

Un regime de definition physique repond a :

```text
dans quel domaine la grandeur joue-t-elle un role ?
```

Un regime d'acces epistemique repond a :

```text
comment cette grandeur est-elle mesuree, inferee, reconstruite, fixee ou bornee ?
```

Exemples :

| Grandeur | Regime physique | Regime d'acces |
|---|---|---|
| G_F | Interaction faible basse energie | Duree de vie du muon, ajustements electrofaibles |
| alpha_s(Q^2) | QCD selon l'echelle de resolution | Extractions multi-processus, evolution RG |
| M_Pl | Gravitation effective basse energie | Grandeur derivee de G_N, hbar et c |
| Lambda | Dynamique globale de l'espace-temps | Inference cosmologique combinee |
| H_0 | Taux actuel d'expansion | Distance ladder, CMB + modele, BAO + calibration |

Regle :

```text
Un accord ou un conflit entre acces ne suffit pas a changer la famille d'une grandeur.
```

### 19. Modes de stabilisation

La stabilisation doit etre decrite explicitement.

Types principaux :

```text
empirique forte
empirique partielle
bornee
reconstruite
modele-dependante
conventionnelle definissante
exacte par composition
non concernee conventionnellement
en tension entre regimes d'acces
```

Exemples :

```text
alpha : stabilisation empirique forte
k_B : stabilisation conventionnelle definissante
R = N_A k_B : exacte par composition
masse absolue neutrino : bornee et modele-dependante selon acces
Lambda : forte dans LambdaCDM, interpretation physique ouverte
H_0 : fortement contraint mais discordant entre routes d'acces
```

Regle :

```text
La stabilisation d'une valeur ne resout pas toujours le statut physique profond de la grandeur.
```

### 20. Tensions de donnees

Une tension de donnees n'est pas une famille.

Elle n'est pas non plus une constante nouvelle.

Elle est une propriete du reseau d'acces et d'inference.

Exemples :

```text
tension H_0
tension S_8
preference DESI pour des modeles d'energie noire dynamique
```

Une tension peut signaler :

```text
systematique non comprise
calibration incomplete
incompatibilite partielle entre jeux de donnees
limite d'un modele
besoin d'une extension phenomenologique
indice possible de nouvelle physique
```

Mais la regle methodologique est :

```text
Tension = propriete d'acces, non categorie taxonomique.
```

### 23. Protocole de classement

Pour classer une nouvelle grandeur, proceder dans cet ordre.

```text
1. Identifier la grandeur exacte.
2. Dire si elle est dimensionnee ou sans dimension.
3. Determiner sa fonction principale.
4. Chercher si une famille superieure suffit.
5. Sous-typer la famille.
6. Ajouter les fonctions transversales sans les confondre avec la famille.
7. Ajouter les fonctions architecturales si la grandeur joue dans une configuration globale.
8. Distinguer regime physique et regime d'acces.
9. Preciser le mode de stabilisation.
10. Dire ce que la grandeur ne dit pas.
```

Regle de priorite :

```text
La fonction principale prime sur les roles secondaires.
```

Exemples :

```text
M_W        : Echelle, meme s'il joue une fonction seuil / validite
Lambda_QCD : Echelle, meme si elle marque un passage de regime
Delta m^2  : Relation spectrale, meme si elle dephase
theta_W    : Orientation, meme si elle depend d'un schema
G_F        : Raccordement, meme s'il implique des relations avec g, M_W et v
```

### 24. Erreurs typiques a eviter

Erreur 1 :

```text
creer une famille des qu'une fonction revient souvent
```

Correction :

```text
tester d'abord si cette fonction est transversale ou architecturale.
```

Erreur 2 :

```text
confondre mesure precise et statut fondamental
```

Correction :

```text
une grandeur peut etre tres precise et rester modele-dependante ou interpretee.
```

Erreur 3 :

```text
confondre architecture et famille
```

Correction :

```text
une architecture articule plusieurs familles ; elle ne remplace pas le noyau taxonomique.
```

Erreur 4 :

```text
classer une tension comme grandeur physique nouvelle
```

Correction :

```text
decrire la tension comme propriete du regime d'acces.
```

Erreur 5 :

```text
forcer toute grandeur cosmologique dans Couplage, Echelle ou Relation
```

Correction :

```text
accepter les fonctions architecturales lorsque la famille principale simple n'est pas pertinente.
```

### 28. Formule de cloture

La v1.1 peut etre resume par quatre formules.

```text
constance = robustesse fonctionnelle situee
famille = fonction principale
fonction transversale = role secondaire ou transversal
architecture = articulation de familles dans un regime
```

Formule finale :

> Une architecture n'ajoute pas une famille ; elle montre comment plusieurs familles deviennent solidaires dans un regime de description.
