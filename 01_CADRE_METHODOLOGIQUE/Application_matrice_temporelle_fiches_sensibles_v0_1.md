# Application de la matrice temporelle aux fiches sensibles v0.1

## Criblage temporel des cas limites deja identifies

### 1. Fonction du document

Ce document applique la matrice temporelle v0.1 aux fiches sensibles deja identifiees par le corpus.

Il prolonge :

```text
01_CADRE_METHODOLOGIQUE/Application_matrice_fiches_sensibles_v0_1.md
01_CADRE_METHODOLOGIQUE/Matrice_temporelle_v0_1.md
01_CADRE_METHODOLOGIQUE/Matrice_criblage_taxonomique_v0_1.md
01_CADRE_METHODOLOGIQUE/Note_synthese_methodologique_v1_3_pre_familial_et_temporalite.md
```

Il ne cree pas de nouvelle famille.

Il ne cree pas non plus une nouvelle architecture.

Objectif :

```text
verifier que les cas sensibles ne confondent pas
temps physique,
echelle,
validite,
acces,
convention,
tension,
ou maturation methodologique.
```

### 2. Perimetre

Les six dossiers sensibles retenus sont ceux du criblage transversal precedent :

```text
1. m_f = y_f v / sqrt(2)
2. k_B et R = N_A k_B
3. alpha_s(Q^2) / Lambda_QCD
4. H_0
5. masse absolue des neutrinos
6. M_Pl et interface cosmologie primordiale
```

La matrice temporelle impose deux dedoublements locaux :

```text
k_B et R n'ont pas exactement la meme trajectoire temporelle ;
alpha_s(Q^2) et Lambda_QCD n'ont pas exactement la meme temporalite.
```

Le document traite donc huit objets dans six dossiers.

### 3. Resultat general

Resultat :

```text
aucun des cas sensibles ne demande une nouvelle famille temporelle.
```

Ils demandent plutot :

```text
1. une identification du type de temporalite ;
2. une variable ou un support explicite ;
3. une distinction entre regime physique et regime d'acces ;
4. une limite d'interpretation formulee.
```

Decision generale :

```text
la temporalite devient un controle de trajectoire,
non une nouvelle case taxonomique.
```

### 5. Cas 1 : `m_f = y_f v / sqrt(2)`

#### 5.1 Matrice temporelle

| Champ | Application |
|---|---|
| Temporalite principale | Constitution |
| Temporalites secondaires | Acces et inference pour les masses et Yukawa |
| Variable ou support | Brisure electrofaible, diagonalisation, regime brise |
| Condition d'installation | `v`, `Y_f` et diagonalisation rendent les masses physiques lisibles |
| Domaine de persistance | Regime electrofaible brise et schema de masses physiques |
| Transition ou limite | Avant brisure, la masse fermionique n'a pas le meme statut |
| Mode d'acces temporel | Mesures de masses, ajustements, reconstruction des Yukawa |
| Trajectoire de stabilisation | Constitution apres brisure, puis stabilisation spectrale |
| Tension eventuelle | Pas de tension temporelle principale ; origine des Yukawa ouverte |
| Effet sur l'architecture | Fort : confirme l'architecture constitutive Saveur-Higgs |
| Limite explicite | Ne donne pas l'origine dynamique des Yukawa |

#### 5.2 Test de confusion

Erreur evitee :

```text
prendre la formule locale m_f = y_f v / sqrt(2)
pour une relation intemporelle isolee.
```

Decision :

```text
temporalite de constitution confirmee.
```

Formule :

> `m_f` est une masse stabilisee dans un regime installe ; sa temporalite principale est celle de la constitution du secteur brise, non celle d'une relation abstraite.

### 6. Cas 2a : `k_B`

#### 6.1 Matrice temporelle

| Champ | Application |
|---|---|
| Temporalite principale | Convention / acces |
| Temporalites secondaires | Role physique thermodynamique conserve |
| Variable ou support | SI, definition du kelvin, realisations thermometriques |
| Condition d'installation | Decision definitionnelle dans l'architecture SI |
| Domaine de persistance | Systeme SI et protocoles de realisation |
| Transition ou limite | Passage d'une valeur mesuree a une valeur definissante |
| Mode d'acces temporel | Absorption conventionnelle puis realisations pratiques |
| Trajectoire de stabilisation | Stabilisation empirique historique -> exactitude SI |
| Tension eventuelle | Pas de tension taxonomique si regime physique et SI restent separes |
| Effet sur l'architecture | Fort : architecture metrologique definitionnelle |
| Limite explicite | L'exactitude SI n'explique pas l'origine physique de la thermodynamique |

#### 6.2 Test de confusion

Erreur evitee :

```text
confondre exactitude numerique par convention
et explication physique de l'energie thermique.
```

Decision :

```text
stabilisation conventionnelle explicite.
```

Formule :

> `k_B` a une temporalite institutionnelle dans le SI, mais son role physique reste thermodynamique.

### 7. Cas 2b : `R = N_A k_B`

#### 7.1 Matrice temporelle

| Champ | Application |
|---|---|
| Temporalite principale | Acces et convention |
| Temporalites secondaires | Composition exacte, acces operationnel thermodynamique |
| Variable ou support | Composition exacte `N_A k_B` dans le SI |
| Condition d'installation | Exactitude definissante de `N_A` et `k_B` |
| Domaine de persistance | Architecture SI et thermodynamique macroscopique |
| Transition ou limite | `R` herite son exactitude ; il ne devient pas definissant primaire |
| Mode d'acces temporel | Exactitude derivee et usage macroscopique |
| Trajectoire de stabilisation | Composition exacte apres definitions SI |
| Tension eventuelle | Aucune si composition et famille sont separees |
| Effet sur l'architecture | Local : composition dans l'architecture metrologique |
| Limite explicite | `R` ne doit pas rebasculer dans Relation par defaut |

#### 7.2 Test de confusion

Erreur evitee :

```text
prendre une composition exacte pour une sortie relationnelle.
```

Decision :

```text
composition exacte avec temporalite derivee.
```

Formule :

> `R` n'a pas la meme temporalite que `k_B` : son exactitude est derivee d'une composition stabilisee par l'architecture SI.

### 8. Cas 3a : `alpha_s(Q^2)`

#### 8.1 Matrice temporelle

| Champ | Application |
|---|---|
| Temporalite principale | Variation ou flot |
| Temporalites secondaires | Acces et schema de renormalisation |
| Variable ou support | `Q^2`, echelle de resolution, schema RG |
| Condition d'installation | QCD et definition d'un couplage courant |
| Domaine de persistance | Domaine ou le schema et l'echelle sont explicites |
| Transition ou limite | Approche du regime non perturbatif |
| Mode d'acces temporel | Extractions multi-processus + evolution RG |
| Trajectoire de stabilisation | Running controle, non instabilite physique |
| Tension eventuelle | Dependances de schema et d'extraction a documenter |
| Effet sur l'architecture | Fort : effectivite par flot de couplage |
| Limite explicite | Le running n'est pas une histoire temporelle de la constante |

#### 8.2 Test de confusion

Erreur evitee :

```text
confondre echelle de resolution et temps historique.
```

Decision :

```text
variation controlee par echelle.
```

Formule :

> `alpha_s(Q^2)` varie avec une variable de resolution ; cette variation est un mode de stabilisation, non une faiblesse de constance.

Decision :

validite de regime confirmee.

### 10. Cas 4 : `H_0`

#### 10.1 Matrice temporelle

| Champ | Application |
|---|---|
| Temporalite principale | Acces et inference |
| Temporalites secondaires | Valeur actuelle dans une evolution cosmologique |
| Variable ou support | `t_0`, routes locales, CMB + modele, BAO + calibration |
| Condition d'installation | Modele cosmologique et definition du taux actuel |
| Domaine de persistance | Cosmologie homogene et isotrope, contexte d'inference declare |
| Transition ou limite | `H_0` n'est pas `H(z)` complet |
| Mode d'acces temporel | Mesure locale, inference precoce, calibrations combinees |
| Trajectoire de stabilisation | Forte contrainte mais divergence inter-acces |
| Tension eventuelle | Tension `H_0` inter-acces, sous stress test |
| Effet sur l'architecture | Fort : sous-reseau fond / expansion et acces |
| Limite explicite | La tension ne prouve pas une variation temporelle directe de `H_0` |

#### 10.2 Test de confusion

Erreurs evitees :

```text
confondre valeur actuelle et fonction d'expansion complete ;
confondre tension d'acces et nouvelle famille ;
confondre divergence de routes avec variation de la constante.
```

Decision :

```text
tension inter-acces documentee.
```

Formule :

> `H_0` est temporellement actuel et epistemiquement pluriel ; sa tension appartient d'abord aux routes d'acces et au modele d'inference.

### 11. Cas 5 : masse absolue des neutrinos

#### 11.1 Matrice temporelle

| Champ | Application |
|---|---|

| Temporalites secondaires | Constitution du secteur neutrino massif |
| Variable ou support | `m_beta`, `Sigma_m_nu`, `m_beta_beta`, masse du plus leger |
| Condition d'installation | Spectre neutrino massif, oscillations, acces heterogenes |

| Transition ou limite | Passage hierarchique / quasi degenere selon ancrage |
| Mode d'acces temporel | Beta decay, cosmologie, double beta sans neutrino sous hypotheses |
| Trajectoire de stabilisation | Bornes, projections non equivalentes, non convergence positive complete |

#### 11.2 Test de confusion

```text
deduire la masse absolue des seuls ecarts oscillatoires ;
assimiler `m_beta`, `Sigma_m_nu` et `m_beta_beta` ;
transformer une borne cosmologique en mesure locale directe.
```

Decision :

```text
trajectoire bornee sous audit.
```

Formule :

> La masse absolue des neutrinos a une temporalite d'acces et de seuil spectral : son ancrage reste borne par plusieurs projections qui ne convergent pas encore en une determination unique.

| Mode d'acces temporel | Grandeur derivee, inferentielle, conventionnelle par choix reduit ou non reduit |

| Tension eventuelle | Pas de tension directe ; completion quantique ouverte |

```text
traiter `M_Pl` comme une frontiere experimentale nette ;
ouvrir trop vite une architecture cosmologique primordiale ;
confondre echelle de validite attendue et theorie de gravitation quantique.
```

Decision :

```text
validite de regime confirmee avec interface prudente.
```

Formule :

> `M_Pl` stabilise une temporalite de validite attendue : il balise une limite de description, sans fournir a lui seul une nouvelle architecture cosmologique.

### 14. Effet sur la suite du travail

Cette application ne force pas la reecriture immediate des fiches sources.

Elle sert plutot de couche de controle.

Decision :

```text
les addenda locaux deviennent necessaires seulement si une reprise de fiche
revele une confusion temporelle non resolue par les syntheses actuelles.
```

Suite logique possible :

```text
1. garder cette application comme reference transversale ;
2. appliquer localement la matrice lors des prochaines reprises de fiches ;
3. ouvrir seulement ensuite une carte consolidee de niveau superieur si le corpus en a besoin.
```

### 15. Formule de cloture

> Les fiches sensibles ne sont pas sensibles parce qu'elles exigent une nouvelle temporalite ; elles le sont parce qu'elles obligent a dire de quel temps, de quelle echelle, de quel acces ou de quelle validite on parle.
