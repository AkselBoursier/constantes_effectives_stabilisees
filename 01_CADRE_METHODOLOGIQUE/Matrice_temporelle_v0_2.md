# Matrice temporelle v0.2

## Scission constitution physique / acces epistemique

### 0. Statut documentaire

```text
statut : protocole actif ;
complete : 01_CADRE_METHODOLOGIQUE/Matrice_temporelle_v0_1.md ;
source critique : 01_CADRE_METHODOLOGIQUE/Fiche_criblage_critiques_lot2_v0_1.md ;
source decisionnelle : 01_CADRE_METHODOLOGIQUE/Addendum_matrice_criblage_lot2_v0_1.md ;
source de precision : 01_CADRE_METHODOLOGIQUE/Note_situation_effective_et_equilibrage_SI_v0_1.md ;
fonction : separer les temps de constitution des temps d'acces.
```

La v0.1 nomme les temporalites.

La v0.2 les scinde lorsque la confusion devient structurante.

Formule directrice :

```text
ce qui rend une grandeur definissable n'est pas toujours ce qui la rend
connue.
```

### 1. Probleme traite

Les critiques du lot 2 montrent une faiblesse possible :

```text
une meme fiche peut melanger constitution physique, acces epistemique,
reconstruction modele-donnees et maturation documentaire.
```

La v0.2 impose donc deux questions separees :

| Axe | Question | Sortie |
|---|---|---|
| Constitution physique | Quelles conditions rendent la grandeur definissable dans un regime ? | Regime, seuil, transition, dependances physiques |
| Acces epistemique | Par quelles routes la grandeur devient-elle mesuree, reconstruite, bornee ou fixee ? | Mesures, modeles, conventions, tensions |

Cette scission ne privilegie pas l'acces contre la constitution physique.

Elle empeche deux confusions symetriques :

```text
prendre l'acces pour l'objet ;
prendre l'objet pour immediatement disponible hors acces.
```

Un troisieme axe peut etre note, mais il ne doit pas etre confondu avec les deux premiers :

| Axe secondaire | Question | Usage |
|---|---|---|
| Maturation documentaire | Comment le statut du dossier s'est-il stabilise dans le corpus ? | Historique methodologique, non preuve physique |

### 1.1 Usage de "situe"

Dans la matrice v0.2, `situe` doit etre lu selon deux niveaux principaux.

```text
situation d'effectuation : le regime ou la grandeur devient definissable ;
situation d'acces : la route par laquelle elle devient disponible au corpus.
```

La situation d'acces ne remplace pas la situation d'effectuation.

Elle la qualifie pour le statut de constance.

### 2. Matrice de constitution physique

La constitution physique decrit le moment ou le statut de la grandeur devient possible dans un regime.

| Champ | Question | Sortie attendue |
|---|---|---|
| Objet | Quelle grandeur ou structure est examinee ? | Symbole, domaine, reseau |
| Regime constituant | Dans quel regime la grandeur devient-elle definissable ? | Electrofaible, QCD, cosmologique, thermodynamique, gravitationnel |
| Condition d'installation | Quel seuil, transition, etat ou cadre la rend possible ? | Brisure, confinement, recombinaison, definition de domaine |
| Dependances physiques | Quelles autres grandeurs ou structures sont requises ? | Couplages, echelles, matrices, champs, etats |
| Domaine de persistance | Ou le statut reste-t-il robuste ? | Domaine d'energie, epoque, regime effectif |
| Variation interne | La grandeur depend-elle d'une variable physique ou d'echelle ? | `Q`, `E`, `z`, `a`, temperature, schema |
| Degradation | Que devient le regime si la condition varie ou disparait ? | Dissolution locale, changement de role, perte d'architecture |
| Limite | Ce que cette constitution ne prouve pas | Origine profonde, statut ontologique, extension hors regime |

### 3. Matrice d'acces epistemique

L'acces epistemique decrit la route par laquelle la grandeur devient disponible pour le corpus.

Il ne remplace pas la constitution physique ou operationnelle de la grandeur.

| Champ | Question | Sortie attendue |
|---|---|---|
| Route d'acces | Comment la grandeur est-elle connue ? | Directe, reconstruite, combinee, bornee, conventionnelle |
| Support donnees / operation | Quel instrument, protocole ou type de donnees intervient ? | Spectroscopie, collision, CMB, BAO, metrologie, fits |
| Modele d'inference | Quel modele conditionne l'extraction ? | Faible, explicite, fort, dominant |
| Degenerescences | Quelles autres grandeurs peuvent compenser ou masquer l'effet ? | Parametres, calibrations, priors, hypotheses |
| Robustesse inter-acces | Les routes convergent-elles ? | Convergente, fragile, divergente, persistante |
| Tension eventuelle | Quel type de desaccord apparait ? | Locale, inter-acces, persistante, point de bascule candidat |
| Substitution | Quelle route ou modele alternatif teste la dependance ? | Acces independant, modele controle, reconstruction alternative |
| Limite | Ce que l'acces ne prouve pas | Variation physique, origine, universalite hors modele |

### 4. Lecture du couplage entre les deux axes

Apres les deux matrices, qualifier le couplage :

| Couplage | Description | Decision |
|---|---|---|
| Faible | La constitution est claire et l'acces la mesure sans reorganiser le statut | Classement possible avec reserve ordinaire |
| Explicite | L'acces reconstruit une grandeur deja definie dans le regime | Mentionner la route d'acces dans la sortie |
| Fort | Le statut effectif depend fortement du modele ou du reseau d'acces | Pas de classement sans qualification d'acces |
| Conflictuel | Plusieurs acces robustes ne convergent pas | Passerelle d'escalade des tensions requise |
| Conventionnel | L'acces fixe la valeur numerique par definition | Separer exactitude numerique et role physique |

Regle :

```text
un couplage fort regime-acces ne fusionne pas les deux axes ;
il oblige a les tenir ensemble dans la decision.
```

### 5. Arbre de decision temporel

| Diagnostic | Sortie prudente |
|---|---|
| Constitution physique claire + acces convergent | Sortie taxonomique possible |
| Constitution claire + acces divergent | Tension d'acces, pas variation physique automatique |
| Constitution modele-dependante + acces modele-dependant | Classement sous modele explicite ou dossier differe |
| Constitution architecturale + acces partiels | Architecture active, decision locale limitee |
| Acces conventionnel + role physique conserve | Exactitude definissante separee du role physique |
| Tension persistante + substitution disponible | Ouvrir un test de substitution |
| Degenerescence inter-parametres active | Ne pas isoler la constante hors reseau |

### 8. Effet sur les prochaines fiches

La v0.2 doit etre employee avant :

```text
une passerelle d'escalade des tensions cosmologiques ;
un test de substitution ;
une fiche sur une grandeur reconstruite par plusieurs routes ;
une sortie taxonomique qui depend d'une temporalite.
```

Elle ne demande pas de modifier immediatement la carte consolidee.

La carte ne devra recevoir ces ajouts qu'apres un cas traversant.

Passerelle produite :

```text
01_CADRE_METHODOLOGIQUE/Passerelle_escalade_tensions_cosmologiques_v0_1.md
```

### 9. Formule de cloture

```text
Une constante peut etre physiquement constituee avant d'etre bien connue,
bien connue sans etre simple, ou exactement fixee sans perdre son role
physique. La matrice temporelle v0.2 sert a ne pas confondre ces trois
situations.
```
