# Reecriture positive du vocabulaire actif v0.1

## Trace genealogique du probleme de classement

### 1. Fonction du document

Ce document repond a un risque devenu visible dans le corpus actif.

Les deux anciens termes vises sont explicites ici en section genealogique :

```text
"famille" et "relation".
```

Note genealogique :

```text
le chantier a d'abord ete intitule
Dissolution_idiomes_famille_relation_v0_1.

Cet intitule reste la trace historique du probleme repere ;
le fichier actif porte maintenant un titre positif.
```

Dans la suite du document, ils sont designes par :

```text
ancien terme A ;
ancien terme B.
```

Le probleme n'est pas seulement lexical.

Il est methodologique :

```text
une categorie que l'on repete pour l'ecarter reste au centre du raisonnement.
```

Objectif :

```text
sortir de la negation,
reformuler positivement les enonces,
et reserver les anciens termes a une archive historique explicite.
```

### 3. Decision centrale

Decision :

```text
le vocabulaire actif doit devenir positif.
```

Les anciens termes ne doivent subsister que dans quatre cas :

```text
1. titre ou nom historique d'un document source ;
2. section de genealogie methodologique ;
3. citation ou rappel explicite d'une formulation archivee.
4. expression technique controlee : famille fonctionnelle,
   seulement comme sortie taxonomique apres test de resistance.
```

Partout ailleurs, l'enonce doit dire directement :

```text
ce qui est stabilise,
ce que la grandeur fait,
comment elle est accessible,
dans quel regime elle opere,
quel role elle prend,
et quelle architecture ou trajectoire elle engage.
```

### 4. Nouveau vocabulaire actif

| Ancien usage actif | Formulation positive privilegiee |
|---|---|
| substantif de classement global | fonction directrice a tester |
| classement candidat | fonction directrice candidate |
| famille fonctionnelle | sortie taxonomique apres test de resistance |
| classement englobant | architecture ou niveau de solidarite |
| classement faible | fonction non prioritaire, a reformuler |
| liaison entre classements | inter-fonctions ou inter-architectures selon contexte |
| tiroir de liaison non qualifie | liaison stabilisee qualifiee, composition, rapport, ecart, equation d'etat |
| adjectif vague de dependance | structure de dependance, normalisation, composition, couplage, acces |
| negation du classement global | relever de la fonction, du role ou de l'architecture nommee |
| exclusion du tiroir de liaison | identifier la forme positive : rapport, ecart, composition, orientation, acces |

Regle :

```text
ne pas dire seulement ce que le cas n'est pas ;
dire d'abord ce qu'il est.
```

### 5. Transformations types

#### 5.1 Architecture

Ancien style :

```text
X n'appartient pas a l'ancien terme A.
```

Style positif :

```text
X est une architecture de solidarite entre fonctions stabilisees dans un regime.
```

#### 5.2 Forme logique

Ancien style :

```text
X ne doit pas etre range dans l'ancien terme B.
```

Style positif :

```text
X stabilise une composition exacte, un rapport normalise, un ecart spectral
ou une equation d'etat selon le cas.
```

#### 5.3 Role contextuel

Ancien style :

```text
Seuil n'appartient pas a l'ancien terme A.
```

Style positif :

```text
Seuil nomme un role de passage, de limite ou d'ancrage dans un regime donne.
```

#### 5.4 Cosmologie

Ancien style :

```text
H_0 est defini par exclusion de deux tiroirs de classement.
```

Style positif :

```text
H_0 est le parametre d'etat actuel du taux d'expansion,
stabilise par des routes d'acces distinctes.
```

#### 5.5 Metrologie

Ancien style :

```text
R = N_A k_B est defini par exclusion d'un tiroir de liaison.
```

Style positif :

```text
R stabilise une composition exacte dans l'architecture SI,
avec un role thermodynamique macroscopique.
```

### 6. Regle de reecriture

Pour toute phrase contenant les anciens termes, appliquer l'ordre suivant.

```text
1. Identifier si l'occurrence est historique ou active.
2. Si elle est historique, la conserver seulement dans une section genealogique.
3. Si elle est active, chercher la formulation positive.
4. Remplacer la negation par la fonction, le role, l'acces, l'architecture ou la trajectoire.
5. Verifier que la phrase ne recree pas l'ancien terme en commentaire defensif.
```

Test de validation :

```text
la phrase reste-t-elle claire si les anciens mots disparaissent ?
```

Si non, la formulation depend encore trop de l'ancien cadre.

suppression des distinctions fonctionnelles ;
effacement de l'historique ;

```text

par documents actifs,

```
