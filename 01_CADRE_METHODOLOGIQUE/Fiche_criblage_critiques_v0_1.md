# Fiche de criblage des critiques v0.1

## Extraction, recoupement et transformation methodologique des huit critiques

### 1. Fonction de la fiche

Cette fiche traite les huit critiques comme une matiere premiere.

Elle ne cherche pas a :

```text
additionner huit avis
adopter toutes les propositions
rejeter les formulations excessives
produire un resume neutre
```

Elle cherche a :

```text
recuperer les problemes souleves
isoler les propositions exploitables
retrouver les idees discretes cachees dans des formulations discutables
articuler les propositions qui se recoupent
transformer les critiques en chantiers methodologiques
```

Regle de lecture :

```text
une critique superficielle peut contenir une proposition utile ;
une proposition fragile peut signaler un vrai probleme ;
un exemple local peut valoir comme regle generale ;
une formulation trop forte peut devenir feconde apres remaniement.
```

### 2. Corpus critique lu

Les huit documents se trouvent dans :

```text
90_Critiques_ constantes_effectives_stabilisees/
```

Documents traites :

```text
Critique 1 : Fiabiliser la hierarchie des constantes physiques
Critique 2 : Architecture des constantes physiques effectives v2
Critique 3 : Architecture dynamique des constantes physiques
Critique 4 : Constantes fantomes des neutrinos
Critique 5 : Architecture documentaire des constantes physiques
Critique 6 : Refondre la taxonomie des constantes physiques
Critique 7 : Taxonomie des constantes comme fonctions
Critique 8 : Tensions cosmologiques et taxonomie des constantes physiques
```

### 3. Resultat central

Les critiques convergent vers une refonte plus profonde que la seule temporalite.

Resultat :

```text
la methode actuelle a stabilise un vocabulaire,
mais elle doit maintenant distinguer plus nettement :

1. la forme logique de ce qui est stabilise ;
2. la fonction physique de la grandeur ;
3. le role contextuel local, transversal ou architectural ;
4. le regime physique ;
5. le regime d'acces ;
6. le couplage modele-acces ;
7. la trajectoire de stabilisation ;
8. le statut documentaire de chaque decision.
```

La correction principale est donc :

```text
ne pas seulement ajouter des cases ;
changer l'ordre des questions.
```

### 4. Methode de criblage

Chaque critique a ete lue selon quatre filtres.

| Filtre | Question |
|---|---|
| Probleme | Quelle fragilite du corpus est signalee ? |
| Proposition | Quelle solution est proposee explicitement ? |
| Remaniement | Quelle version prudente peut etre gardee ? |
| Transposition | L'exemple local vaut-il pour un autre niveau de la methode ? |

Les propositions sont classees en quatre statuts.

| Statut | Sens |
|---|---|
| A integrer | Proposition robuste et compatible avec la methode |
| A remanier | Idee utile mais formulation trop forte ou trop locale |
| A tester | Hypothese prometteuse mais non encore stabilisee |
| A refuser comme telle | Proposition risquee, mais pouvant signaler un probleme reel |

### 5. Critique 1 : hierarchie documentaire et boucle descendante

Probleme signale :

```text
le flux documentaire est surtout ascendant :
fiches locales -> syntheses -> cartes.
```

Les consolidations ne redescendent pas encore sous forme de contraintes explicites sur les nouvelles fiches.

Proposition brute :

```text
chaque consolidation active doit generer un gabarit contraignant de preuve locale.
```

Idee recuperable :

```text
une consolidation ne doit pas seulement stocker une decision ;
elle doit produire des obligations pour les fiches futures.
```

Remaniement :

```text
creer des gabarits derives des architectures,
mais ne pas les rendre mecaniquement identiques pour tous les secteurs.
```

Statut :

```text
A integrer.
```

Effet methodologique :

```text
ajouter une boucle descendante :
architecture consolidee -> matrice locale obligatoire -> nouvelle fiche.
```

### 6. Critique 2 : architecture ou simple juxtaposition ?

Probleme signale :

```text
la matrice d'architecture peut valider trop facilement des ensembles heterogenes.
```

Proposition brute :

```text
imposer une regle de causalite stricte et transformer la matrice v1.2 en stress test eliminatoire.
```

Idee recuperable :

```text
une architecture ne doit pas etre reconnue parce qu'un meme acces ou un meme tableau regroupe plusieurs grandeurs.
```

Remaniement :

```text
remplacer "causalite stricte" par "solidarite specifiee".
```

Raison :

```text
toutes les architectures ne sont pas causales au meme sens.
Le SI est definitionnel.
La cosmologie est inferentielle.
L'effectif basse energie est souvent un raccordement de validite.
Saveur-Higgs est constitutif.
```

Decision proposee :

```text
chaque architecture doit dire quel type de solidarite elle revendique :
constitutive, definitionnelle, validite, inferentielle, reconstructive, operationnelle.
```

Statut :

```text
A integrer apres remaniement.
```

### 7. Critique 3 : trajectoire de stabilisation

Probleme signale :

```text
les niveaux de stabilisation sont encore trop statiques.
```

Exemple :

```text
H_0 n'est pas simplement "fortement contraint mais discordant".
Il suit une trajectoire : convergence locale forte, divergence inter-reseaux.
```

Proposition brute :

```text
remplacer les etats de stabilisation par une trajectoire de stabilisation.
```

Idee recuperable :

```text
la stabilisation doit decrire un mouvement, non seulement un niveau.
```

Remaniement :

```text
conserver le niveau actuel,
mais ajouter un champ "trajectoire de stabilisation".
```

Exemples de trajectoires :

```text
convergence empirique
divergence entre acces
stabilisation bornee
stabilisation absorbee par convention
reconstruction modele-dependante
blocage par degenerescence de parametres
```

Statut :

```text
A integrer.
```

### 8. Critique 4 : neutrinos, projection et grandeurs absentes de l'acces

Probleme signale :

```text
les neutrinos ne sont pas seulement un cycle de plus ;
ils imposent une rupture entre constance-valeur et constance differentielle.
```

Propositions brutes :

```text
constance differentielle
constance de projection
constantes fantomes pour les phases de Majorana
opposition PMNS / CKM
role dans l'amplitude de transition
manifestation phenomenologique
```

Idees recuperables :

```text
Delta m^2 ne stabilise pas une propriete d'objet isole,
mais un ecart spectral qui devient moteur de phase dans un regime oscillatoire.

PMNS et CKM peuvent rester dans une meme famille Orientation,
mais leur manifestation phenomenologique doit etre distinguee.

Certaines grandeurs structurantes peuvent etre absentes d'un regime d'acces donne.
```

Remaniements :

```text
ne pas adopter officiellement "constantes fantomes" comme categorie.
Garder l'idee sous forme plus rigoureuse :
"parametres structurants hors acces direct du regime considere".
```

Statut :

```text
A integrer partiellement.
```

Effet methodologique :

```text
ajouter dans les fiches un champ :
"manifestation phenomenologique".
```

### 9. Critique 5 : gouvernance documentaire et statuts critiques

Probleme signale :

```text
"consolidation", "reference active" et "a produire" sont encore trop peu normatifs.
```

Proposition brute :

```text
une consolidation doit avoir une duree de vie limitee et un mandat de montee en reference.
```

Idee recuperable :

```text
un statut documentaire doit dire ce qu'un document autorise, bloque ou oblige.
```

Remaniement :

```text
remplacer la logique passive "a produire" par une logique de criticite :
a produire-bloquant,
a produire-methodologique,
a produire-documentaire,
a produire-administratif.
```

Statut :

```text
A integrer.
```

Effet immediat :

```text
Architecture_saveur_Higgs_notes.md a ete identifiee comme dette bloquante
avant une grande carte consolidee temporelle.
Cette dette est levee par la note Markdown active produite ensuite.
```

### 10. Critique 6 : Relation, roles contextuels et grilles orthogonales

Probleme signale :

```text
la famille Relation est trop large,
car toute constante est relationnelle d'une maniere ou d'une autre.
```

Proposition brute :

```text
supprimer la famille Relation
et redistribuer ses membres vers des familles plus dynamiques.
```

Idee recuperable :

```text
Relation confond une forme logique generale avec une fonction taxonomique.
```

Remaniement central :

```text
introduire un rang pre-familial :
forme logique de stabilisation.
```

Ce rang repond a :

```text
qu'est-ce qui est stabilise formellement ?
```

Exemples :

```text
valeur
rapport
ecart
composition
equation d'etat
fonction d'echelle
matrice
phase
projection
observable reconstruite
condition initiale
convention definissante
```

Ensuite seulement vient la question :

```text
que fait la grandeur dans le regime considere ?
```

Statut :

```text
A integrer.
```

Decision provisoire :

```text
Relation ne doit plus etre utilisee comme famille par defaut.
Elle doit etre soit retrogradee,
soit maintenue seulement sous des formes strictement qualifiees et non tautologiques.
```

### 11. Critique 7 : test de resistance taxonomique et reseau de tension

Probleme signale :

```text
la retrogradation de certaines familles peut sembler intuitive ou arbitraire
si aucun test formel ne l'encadre.
```

Proposition brute :

```text
une famille superieure doit passer un test d'autonomie fonctionnelle totale.
```

Idee recuperable :

```text
une famille doit avoir une fonction non reductible a un autre rang.
```

Remaniement :

```text
ne pas exiger une autonomie totale abstraite,
mais un test de non-reduction fonctionnelle.
```

Question du test :

```text
si on retire cette famille,
la fonction qu'elle nomme est-elle encore correctement decrite
par une autre famille, un role contextuel, une forme logique ou une architecture ?
```

Si oui :

```text
retrograder ou redistribuer.
```

Si non :

```text
conserver comme famille candidate.
```

Autre idee recuperable :

```text
les architectures doivent parfois etre representees comme reseaux de dependance,
non seulement comme tableaux.
```

Statut :

```text
A integrer.
```

### 12. Critique 8 : tensions cosmologiques et vecteur diachronique

Probleme signale :

```text
les tensions cosmologiques sont trop passives si elles restent seulement des proprietes d'acces.
```

Proposition brute :

```text
au-dessus d'un seuil statistique, une tension devient indicateur de rupture de validite.
```

Idee recuperable :

```text
une tension persistante peut devenir un signal actif pour tester la validite d'une architecture.
```

Remaniement :

```text
ne pas automatiser la reclassification par un seuil unique.
Ajouter un statut d'activation des tensions.
```

Niveaux possibles :

```text
tension locale
tension inter-acces
tension persistante
tension modele-dependante
tension candidate a rupture de validite
```

Statut :

```text
A tester.
```

Autre idee forte :

```text
vecteur diachronique :
reconstruction inferentielle -> stabilisation empirique -> ancrage architectural -> absorption conventionnelle.
```

Remaniement :

```text
ne pas l'appliquer comme trajectoire universelle,
mais l'utiliser comme profil possible de maturation.
```

Statut :

```text
A integrer dans la methode temporelle.
```

### 13. Convergences fortes entre critiques

Les critiques convergent sur dix points.

#### 13.1 La famille Relation est le point le plus fragile

Convergence :

```text
Relation est trop generale si elle reste au meme rang que Couplage ou Orientation.
```

Decision de criblage :

```text
creer un rang pre-familial de forme logique de stabilisation.
```

Effet :

```text
les rapports, compositions, ecarts et equations cessent d'etre automatiquement des "relations" au sens familial.
```

#### 13.2 La methode a besoin d'un test de non-reduction

Convergence :

```text
Seuil, Fond et Relation posent le meme probleme sous des formes differentes :
quel critere decide le rang d'une categorie ?
```

Decision de criblage :

```text
introduire un test de resistance taxonomique.
```

#### 13.3 Les fonctions transversales et architecturales sont trop separees

Convergence :

```text
la distinction actuelle peut alourdir la methode.
```

Mais la suppression pure et simple serait dangereuse.

Remaniement :

```text
creer le niveau "role contextuel",
avec un champ de portee :
local, transversal, architectural.
```

Ainsi :

```text
Seuil = role contextuel transversal.
Fond = role contextuel architectural.
Validite = role contextuel transversal ou architectural selon le cas.
Observable reconstruite = role contextuel architectural.
```

#### 13.4 Physique et acces doivent etre deux grilles orthogonales

Convergence :

```text
le regime physique et le regime d'acces ne doivent pas etre melanges,
mais ils ne doivent pas non plus etre separes comme s'ils etaient independants.
```

Decision de criblage :

```text
maintenir deux axes :
axe physique
axe epistemique / acces

ajouter un champ :
couplage modele-acces.
```

#### 13.5 L'architecture doit etre testee, pas seulement reconnue

Convergence :

```text
la matrice d'architecture doit pouvoir refuser,
eclater ou reclasser une architecture candidate.
```

Decision :

```text
transformer la matrice d'architecture en stress test.
```

#### 13.6 La cosmologie doit etre re-ouverte prudemment

Convergence :

```text
l'architecture cosmologique est utile,
mais peut-etre trop englobante.
```

Tests a mener :

```text
cosmologie primordiale
cosmologie tardive
architecture de reconstruction
tensions comme signaux actifs
couplage modele-acces
```

Statut :

```text
A tester, sans casser encore l'architecture cosmologique generale.
```

#### 13.7 La temporalite doit devenir un axe officiel

Convergence :

```text
trajectoire de stabilisation,
vecteur diachronique,
cycle de vie,
transition de regime,
absorption conventionnelle.
```

Decision :

```text
la note de revision temporelle est confirmee,
mais elle doit etre combinee avec le rang pre-familial.
```

#### 13.8 Les tensions ne doivent pas rester purement passives

Convergence :

```text
traiter les tensions seulement comme "proprietes d'acces" peut etre trop faible.
```

Remaniement :

```text
tension = propriete d'acces au depart,
mais peut devenir indicateur de test de validite si elle est persistante,
inter-acces, robuste aux modeles et quantitativement forte.
```

#### 13.9 Les architectures doivent etre representees comme reseaux

Convergence :

```text
les tableaux montrent les elements,
mais pas les dependances.
```

Decision :

```text
ajouter des cartes de dependance :
si l'on retire X, qu'est-ce qui perd son sens ?
```

#### 13.10 La gouvernance documentaire est une condition methodologique

Convergence :

```text
un corpus mal gouverne affaiblit la theorie.
```

Decision :

```text
introduire des statuts documentaires operatoires :
reference active,
revision active,
consolidation active,
preuve locale,
archive historique,
a produire-bloquant,
a produire-administratif.
```

### 14. Idees discretes a ne pas perdre

Certaines idees sont moins centrales mais fecondes.

#### 14.1 Robustesse de l'acces

Ajouter :

```text
robustesse du regime d'acces
```

Questions :

```text
l'acces est-il direct, reconstruit, conventionnel, combine, fragile, en tension ?
```

#### 14.2 Couplage modele-acces

Surtout en cosmologie :

```text
les donnees ne suffisent pas ;
le modele d'inference transforme ce qui est accessible.
```

Champ propose :

```text
modele d'inference dominant
```

#### 14.3 Manifestation phenomenologique

Utile pour distinguer :

```text
CKM : orientation a manifestation transitionnelle.
PMNS : orientation a manifestation oscillatoire.
Delta m^2 : ecart spectral a manifestation dephasante.
```

#### 14.4 Stabilisation absorbee

Pour le SI :

```text
stabilisation empirique ancienne -> absorption conventionnelle.
```

Ce terme est plus precis que "exact par convention" lorsqu'on decrit la trajectoire.

#### 14.5 Architecture de reconstruction

La cosmologie peut etre lue comme :

```text
architecture de reconstruction,
ou les tensions ne sont pas des accidents externes,
mais des tests internes de robustesse.
```

#### 14.6 Critere de sortie des consolidations

Une consolidation active devrait dire :

```text
ce qu'elle stabilise
ce qu'elle oblige
ce qu'elle doit mettre a jour
quand elle reste locale
quand elle monte en reference
```

#### 14.7 Dette documentaire bloquante

Toute lacune n'a pas le meme poids.

Exemple :

```text
note Markdown Saveur-Higgs = dette structurante levee ensuite.
README general = dette utile mais non bloquante.
```

#### 14.8 Question de retrait

Pour une architecture :

```text
si l'on retire cette grandeur,
qu'est-ce qui devient indefinissable,
inoperant ou mal classe ?
```

Cette question est plus efficace qu'une simple liste de familles mobilisees.

### 15. Propositions a ne pas adopter telles quelles

#### 15.1 Causalite stricte

Probleme :

```text
utile pour Saveur-Higgs ou certaines architectures effectives,
mais trop stricte pour le SI ou les architectures inferentielles.
```

Remaniement :

```text
solidarite specifiee plutot que causalite unique.
```

#### 15.2 Suppression brute de Relation

Probleme :

```text
risque de perdre les cas ou une liaison est vraiment l'objet stabilise.
```

Remaniement :

```text
deplacer la relationalite generale au rang pre-familial,
puis tester les anciens cas Relation un par un.
```

#### 15.3 Fusion totale transversal / architectural

Probleme :

```text
risque de perdre la difference de portee.
```

Remaniement :

```text
role contextuel + portee locale / transversale / architecturale.
```

#### 15.4 Tension reclassante automatique

Probleme :

```text
un seuil statistique unique ne suffit pas a changer le statut d'une grandeur.
```

Remaniement :

```text
statut d'activation des tensions,
avec conditions cumulatives.
```

#### 15.5 Vecteur diachronique unique

Probleme :

```text
toutes les grandeurs ne passent pas par reconstruction -> stabilisation -> ancrage -> convention.
```

Remaniement :

```text
trajectoire de stabilisation a profils multiples.
```

### 16. Nouvelle architecture methodologique proposee

Les critiques suggerent une methode en six rangs.

| Rang | Nom | Question |
|---|---|---|
| 0 | Forme logique de stabilisation | Qu'est-ce qui est stabilise formellement ? |
| 1 | Famille fonctionnelle | Que fait principalement la grandeur dans ce regime ? |
| 2 | Role contextuel | Quel role local, transversal ou architectural joue-t-elle ? |
| 3 | Axes physique / acces | Dans quel regime opere-t-elle et par quel acces est-elle connue ? |
| 4 | Architecture / reseau | Quelles dependances solidaires produit-elle ? |
| 5 | Temporalite / trajectoire | Comment son statut s'installe, varie, persiste ou se transforme ? |

Cette structure corrige trois confusions :

```text
Relation n'est plus un fourre-tout.
Les roles transversaux et architecturaux ne sont plus des etages trop rigides.
Le temps ne remplace pas la taxonomie ; il decrit la trajectoire de stabilisation.
```

### 17. Rang 0 : forme logique de stabilisation

Le rang 0 doit etre introduit avant la famille.

Question :

```text
quelle forme prend la stabilisation ?
```

Liste provisoire :

| Forme logique | Exemple | Attention |
|---|---|---|
| Valeur | c, h, e dans le SI | Peut etre conventionnelle ou physique selon le contexte |
| Rapport | m_p / m_e, Omega_i | Ne suffit pas a classer la fonction |
| Ecart | Delta m^2 | Peut devenir moteur de dephasage |
| Composition | R = N_A k_B, K_J = 2e/h | Exactitude derivee possible |
| Equation ou relation d'etat | w | Peut etre parametre phenomenologique |
| Fonction d'echelle | alpha_s(Q^2), alpha_G(E) | Exige variable et schema |
| Matrice | CKM, PMNS, Yukawa | Demande manifestation phenomenologique |
| Phase | delta_CP, phases de Majorana | Acces parfois indirect ou absent |
| Observable reconstruite | sigma_8, S_8 | Depend du modele et des routes d'acces |
| Condition initiale | A_s, n_s | Acces tardif a une structure primitive |
| Convention definissante | k_B, N_A, h, e | Ne dit pas l'origine physique |

Regle :

```text
le rang 0 ne classe pas encore la fonction ;
il identifie la forme de ce qui est stabilise.
```

### 18. Re-traitement de l'ancienne famille Relation

Les anciens cas Relation doivent etre repris.

| Ancien cas | Rang 0 | Classement a retester |
|---|---|---|
| m_p / m_e | Rapport stratifie | Hierarchie d'echelles ou relation stratifiee ? |
| R = N_A k_B | Composition exacte | Convention SI + composition |
| Delta m^2 | Ecart spectral | Dynamique de phase / dephasage oscillatoire |
| m_f = y_f v / sqrt(2) | Relation constitutive | Architecture Saveur-Higgs, masse comme stabilisation de regime |
| Omega_i | Rapport normalise | Densite normalisee dans architecture cosmologique |
| w | Relation d'etat | Role contextuel d'etat cosmologique |

Decision provisoire :

```text
Relation cesse d'etre la reponse par defaut.
Elle devient un probleme a decomposer :
forme logique + fonction physique + role contextuel.
```

### 19. Test de resistance taxonomique

Avant de creer ou maintenir une famille, appliquer ce test.

```text
1. La categorie nomme-t-elle une fonction principale ?
2. Cette fonction est-elle irreductible a une forme logique ?
3. Est-elle irreductible a un role contextuel ?
4. Est-elle irreductible a une architecture ?
5. Peut-elle classer plusieurs cas heterogenes sans les ecraser ?
6. Peut-on donner un contre-exemple qui la ferait devenir fourre-tout ?
7. Que perd-on si on la supprime ?
8. Que gagne-t-on si on la retrograde ?
```

Sorties possibles :

```text
famille conservee
famille renommee
role contextuel
forme logique
architecture
archive taxonomique
```

### 20. Stress test d'architecture

Avant de valider une architecture, appliquer ce test.

```text
1. Quel type de solidarite est revendique ?
2. Quelles grandeurs sont necessaires ?
3. Que devient chaque grandeur si l'une des autres est retiree ?
4. La solidarite est-elle physique, definitionnelle, inferentielle, operationnelle ou documentaire ?
5. L'acces commun suffit-il vraiment a etablir une architecture ?
6. Faut-il eclater l'architecture en sous-architectures ?
7. La tension interne signale-t-elle une fragilite d'acces ou une rupture de validite possible ?
```

Sorties possibles :

```text
architecture confirmee
architecture locale
architecture eclatee
architecture de reconstruction
simple regroupement documentaire
```

| Champ | Question |
|---|---|

| Forme logique | Valeur, rapport, ecart, composition, matrice, phase, observable, condition initiale ? |
| Famille fonctionnelle | Que fait principalement la grandeur ? |
| Role contextuel | Role local, transversal ou architectural ? |
| Portee du role | Locale, transversale, architecturale, documentaire ? |
| Regime physique | Dans quel domaine opere-t-elle ? |
| Regime d'acces | Comment est-elle connue ? |
| Couplage modele-acces | Quel modele transforme ou conditionne l'acces ? |
| Robustesse de l'acces | Directe, reconstruite, combinee, fragile, conventionnelle, en tension ? |
| Manifestation phenomenologique | Comment la grandeur se manifeste-t-elle ? |
| Trajectoire de stabilisation | Convergence, divergence, absorption conventionnelle, reconstruction, blocage ? |
| Tension eventuelle | Passive, inter-acces, persistante, candidate a rupture ? |
| Architecture eventuelle | Quelle solidarite specifiee ? |
| Test de retrait | Que perd-on si cette grandeur est retiree ? |
| Limite | Ce que le classement ne dit pas |


### 24. Formule de cloture

Les critiques ne demandent pas d'abandonner le projet.

Elles demandent de changer le niveau de certaines questions.

Formule :

```text
avant de demander a quelle famille appartient une constante,
il faut demander quelle forme de stabilisation elle prend,
dans quel regime elle opere,
par quel acces elle est connue,
et quelle trajectoire de stabilisation elle suit.
```

Formule finale :

> La taxonomie ne doit pas seulement ranger des constantes ; elle doit montrer comment une forme stabilisee devient fonction, comment une fonction devient role, comment des roles deviennent architecture, et comment l'ensemble se stabilise ou se fragilise dans le temps.
