# Revision de fond v0.1 - dimension temporelle

## Reorganisation methodologique et theorique du corpus

### 1. Fonction de cette note

Cette note ouvre une revision de fond.

Elle ne remplace pas encore :

```text
01_CADRE_METHODOLOGIQUE/Note_synthese_methodologique_v1_2.md
05_CARTES_ET_SYNTHESES/Carte_consolidee_v1_1.md
05_CARTES_ET_SYNTHESES/Synthese_architectures_inter_familles_v1_0.md
```

Elle examine une limite devenue visible apres la stabilisation du corpus :

```text
la methode classe bien les grandeurs,
mais elle decrit encore trop peu la dynamique par laquelle elles deviennent constantes,
restent constantes,
changent de statut,
ou deviennent accessibles.
```

La question directrice est donc :

> Comment introduire une dimension temporelle sans affaiblir la rigueur taxonomique acquise ?

### 2. Diagnostic general

Le corpus a bien avance.

Il a stabilise :

```text
six familles superieures
cinq fonctions transversales
des fonctions architecturales
quatre architectures inter-familles
une matrice canonique
un index documentaire
```

Mais cette stabilisation produit un effet secondaire.

La carte actuelle ressemble encore a une table de positions :

```text
grandeur -> famille -> fonction -> regime -> acces -> stabilisation -> architecture
```

Cette forme est utile.

Elle evite :

```text
l'inflation de categories
la confusion famille / architecture
la confusion exactitude SI / origine physique
la confusion tension / verdict physique
```

Mais elle rend moins visible une dimension centrale :

```text
une constante effective stabilisee n'est pas seulement placee dans une case ;
elle se constitue, se maintient, se transforme ou devient accessible dans un processus.
```

### 3. Ce que la lecture du corpus montre

La dimension temporelle n'est pas absente.

Elle est deja presente, mais dispersee.

#### 3.1 Dans le brief initial

Le brief insistait deja sur :

```text
regimes physiques determines
genealogie du projet
phases de construction du corpus
cycle effectif basse energie
prochaine fiche logique
```

Il contenait aussi une sequence de phases :

```text
corpus initial
extension electrofaible, faible, gravitationnelle et cosmologique
neutrinos
thermodynamique et metrologie
CKM et orientation
Saveur-Higgs
cartes consolidees
cycle effectif basse energie
```

Cette genealogie est documentaire, mais elle reflete aussi une maturation conceptuelle.

#### 3.2 Dans la methode initiale

La methode v0.1 disait deja :

```text
une constante physique n'est pas seulement une valeur stable ;
elle est une grandeur dont la stabilite prend sens dans un regime physique determine.
```

Elle introduisait aussi :

```text
dependance d'echelle
mode d'acces experimental
degre de stabilisation
regime de definition
questions ouvertes
```

Cette note portait donc deja une dynamique, mais sans la formaliser comme axe autonome.

#### 3.3 Dans la note sur les regimes de constance

La note de cadrage v0.2 contenait la formulation la plus temporelle :

```text
a partir de quand une grandeur devient-elle physiquement definissable
dans l'histoire de l'univers ?
```

Elle introduisait aussi :

```text
brisure de symetrie
confinement
refroidissement cosmologique
formation des noyaux
constitution des atomes
passage a un regime basse energie
```

Ce point est important.

La temporalite n'est pas seulement une question de date ou d'histoire documentaire.

Elle est une condition de definissabilite physique.

#### 3.4 Dans les notes critiques

La note de compression montrait deja que certaines categories etaient trop hautes.

Exemples :

```text
Seuil etait une fonction de passage, non une famille.
Fond etait une fonction architecturale, non une famille.
Relation devait survivre seulement comme famille qualifiee.
```

Ces decisions sont dynamiques :

```text
une categorie peut naitre comme hypothese,
etre testee,
puis etre retrogradee ou stabilisee.
```

La methode elle-meme a donc une temporalite.

#### 3.5 Dans les cycles physiques

Les cycles recents montrent plusieurs formes de temps.

```text
alpha_s(Q^2) / Lambda_QCD : flot de couplage et changement de regime.
M_W : validite d'une description basse energie.
M_Pl : limite de validite attendue, non frontiere nette.
Lambda : terme de fond fortement stabilise, statut physique ouvert.
H_0 : parametre d'etat actuel.
w : relation d'etat, eventuellement dynamique dans certaines parametrisations.
A_s / n_s : conditions initiales perturbatives.
sigma_8 / S_8 : observables reconstruites tardives.
c / h / e : valeurs exactes par convention SI, roles physiques conserves.
```

Ces exemples ne demandent pas une nouvelle famille.

Ils demandent un axe temporel transversal.

### 4. Decision centrale

La temporalite ne doit pas devenir une famille superieure.

Elle ne doit pas non plus devenir une cinquieme architecture inter-familles.

Decision :

```text
Temporalite = axe methodologique transversal.
```

Elle repond a une question differente des autres niveaux.

```text
Famille : que fait principalement la grandeur ?
Fonction transversale : quel role secondaire ou transversal joue-t-elle ?
Fonction architecturale : quel role joue-t-elle dans une configuration globale ?
Architecture : comment plusieurs fonctions deviennent-elles solidaires ?
Temporalite : comment la grandeur entre-t-elle dans son statut, le conserve-t-elle ou le transforme-t-elle ?
```

Formule :

```text
la temporalite n'ajoute pas une case ;
elle decrit le processus de stabilisation d'une case.
```

### 5. Pourquoi l'axe temporel est necessaire

Sans axe temporel, la methode risque trois aplatissements.

#### 5.1 Aplatissement des regimes

Une grandeur peut avoir un statut seulement apres installation d'un regime.

Exemples :

```text
m_f apres brisure electrofaible.
Delta m^2 dans le regime oscillatoire.
Lambda_QCD dans le passage perturbatif / non perturbatif.
Omega_i dans un modele cosmologique donne.
```

Si on ne dit pas quand ou sous quelle condition le regime s'installe, la classification devient trop statique.

#### 5.2 Aplatissement des stabilisations

Toutes les stabilisations ne sont pas du meme ordre.

```text
empirique forte
empirique partielle
bornee
reconstruite
modele-dependante
conventionnelle definissante
exacte par composition
en tension entre acces
```

Ces modes ont une temporalite differente.

Une valeur peut etre :

```text
mesuree progressivement
fixee par convention a une date metrologique
reconstruite par un modele
stabilisee dans un domaine mais ouverte dans son interpretation
```

#### 5.3 Aplatissement des architectures

Les quatre architectures n'ont pas le meme rapport au temps.

```text
Saveur-Higgs : constitution d'un secteur apres brisure et diagonalisation.
Metrologique SI : fixation conventionnelle et realisations pratiques.
Effective basse energie : validite par domaine, echelle et changement de regime.
Cosmologique : histoire de l'univers, modeles d'evolution et routes d'inference.
```

Les traiter uniquement comme quatre colonnes paralleles perd une partie de leur difference interne.

### 6. Les cinq temporalites a distinguer

La revision propose de distinguer cinq temporalites.

#### 6.1 Temporalite de constitution

Question :

```text
quelles conditions doivent etre installees pour que la grandeur ait le statut qu'on lui attribue ?
```

Exemples :

```text
brisure electrofaible pour les masses fermioniques
confinement pour la masse du proton et Lambda_QCD
regime oscillatoire pour Delta m^2
modele LambdaCDM pour certains parametres cosmologiques
reforme du SI pour l'exactitude definissante de h, e, k_B, N_A
```

Cette temporalite ne dit pas seulement "avant / apres".

Elle dit :

```text
condition de definissabilite.
```

#### 6.2 Temporalite de variation ou de flot

Question :

```text
la grandeur depend-elle d'une echelle, d'un schema, d'un temps cosmologique ou d'une variable d'evolution ?
```

Exemples :

```text
alpha_s(Q^2) court avec l'echelle.
alpha_G(E) se lit par rapport a une echelle d'energie.
w peut etre constant dans LambdaCDM ou dynamique dans w0waCDM.
Omega_i peut changer de signification selon l'epoque cosmologique consideree.
H(z) est dynamique, tandis que H_0 est une valeur actuelle.
```

Regle :

```text
une grandeur qui varie avec une variable controlee n'est pas moins rigoureuse ;
elle exige une indexation explicite.
```

#### 6.3 Temporalite de transition ou de validite

Question :

```text
ou une description cesse-t-elle d'etre suffisante ou change-t-elle de regime ?
```

Exemples :

```text
M_W comme limite de validite de la theorie de Fermi.
Lambda_QCD comme changement perturbatif / non perturbatif.
M_Pl comme limite attendue, non frontiere nette.
Seuil comme role d'une echelle ou d'une relation.
Validite comme domaine ou une description reste applicable.
```

Cette temporalite est centrale pour l'architecture effective basse energie.

#### 6.4 Temporalite d'acces et d'inference

Question :

```text
comment la grandeur devient-elle connue, reconstruite, comparee ou mise en tension ?
```

Exemples :

```text
H_0 par distance ladder, CMB + modele, BAO + calibration.
S_8 par lentille faible, CMB + modele, amas, 3x2pt.
A_s par CMB et degenerescence avec tau.
G_N par mesures difficiles, avec stabilisation empirique moins precise.
constantes SI par realisations pratiques.
```

Cette temporalite n'est pas le temps physique de la grandeur.

Elle est le temps de son acces, de sa reconstruction et de sa comparaison.

#### 6.5 Temporalite de maturation methodologique

Question :

```text
comment la categorie elle-meme s'est-elle stabilisee dans le corpus ?
```

Exemples :

```text
Seuil : famille provisoire -> fonction transversale.
Fond : famille fragile -> fonction architecturale.
Relation : famille faible -> famille conservee mais qualifiee.
Saveur-Higgs : secteur disperse -> architecture constitutive.
Metrologie SI : cas k_B / N_A / R -> architecture definitionnelle etendue.
```

Cette temporalite appartient au corpus, non directement a la physique.

Mais elle est methodologiquement importante, car elle montre comment les decisions ont ete testees.

### 7. Ne pas confondre les temporalites

Le point critique est de ne pas confondre :

```text
temps cosmologique
echelle d'energie
temps d'acces experimental
temps institutionnel d'une convention
temps genealogique du corpus
```

Ces temporalites peuvent interagir, mais elles ne sont pas equivalentes.

Exemples :

```text
Le running de alpha_s n'est pas une evolution historique de la constante.
La definition SI de h ne donne pas l'origine physique de h.
La tension H_0 n'est pas une evolution temporelle directe de H_0.
La retrogradation de Seuil est une maturation methodologique, non un fait physique.
```

Regle :

```text
toute mention temporelle doit dire de quel temps elle parle.
```

### 8. Proposition de matrice temporelle

Chaque fiche importante devrait ajouter une matrice temporelle minimale.

| Champ | Question | Risque controle |
|---|---|---|
| Condition d'installation | Qu'est-ce qui rend la grandeur definissable ? | Supposer une constance hors regime |
| Variable d'evolution | Temps, redshift, echelle Q/E, schema, jeu de donnees, convention ? | Confondre les types de temps |
| Domaine de persistance | Dans quel domaine le statut reste-t-il stable ? | Universaliser trop vite |
| Transition ou limite | Ou le statut change-t-il ou devient-il insuffisant ? | Reifier un seuil |
| Mode d'acces temporel | Acces direct, retrospectif, reconstruit, borne, combine ? | Confondre mesure et inference |
| Maturation de la stabilisation | Forte, partielle, bornee, conventionnelle, reconstruite, en tension ? | Donner la meme maturite a tous les cas |
| Effet sur l'architecture | La temporalite solidarise-t-elle plusieurs fonctions ? | Rater une architecture |
| Limite explicite | Ce que cette dynamique ne permet pas de conclure | Transformer un indice en verdict |

Cette matrice doit completer la matrice v1.2.

Elle ne doit pas la remplacer.

### 9. Effet sur les six familles

#### 9.1 Couplage

Le couplage doit etre lu avec deux questions temporelles.

```text
est-il stable dans un regime ?
court-il avec une echelle ?
```

Exemples :

```text
alpha : couplage sans dimension fortement stabilise, tests de variation.
alpha_s(Q^2) : couplage courant, indexe a une echelle.
alpha_G(E) : couplage gravitationnel effectif de regime.
Y_f : couplages de texture, lisibles dans le regime electrofaible brise.
```

#### 9.2 Echelle

Une echelle n'est pas seulement une grandeur de taille.

Elle peut etre :

```text
condition de regime
limite de validite
seuil faible ou fort
trace d'une dynamique
```

Exemples :

```text
v installe l'echelle electrofaible brisee.
M_W indique la validite de la theorie de Fermi.
Lambda_QCD indique une dynamique de confinement.
M_Pl indique une validite attendue.
```

#### 9.3 Relation

La famille Relation est naturellement temporelle lorsqu'elle relie :

```text
des regimes
des niveaux
des spectres
des phases
des compositions exactes
des etats cosmologiques
```

Mais elle doit rester qualifiee.

La temporalite ne dispense pas de sous-typer Relation.

#### 9.4 Raccordement

Raccordement est la famille la plus explicitement dynamique.

Elle dit :

```text
comment une description effective se relie a une description plus complete,
ou comment un niveau condense ce qui est resolu ailleurs.
```

Elle doit toujours inclure :

```text
domaine de validite
niveau de description
limite de raccordement
ce que le raccordement ne resout pas
```

#### 9.5 Orientation

Orientation porte une temporalite locale par ses manifestations.

Exemples :

```text
PMNS : orientation de saveur a manifestation oscillatoire.
CKM : orientation de saveur a manifestation transitionnelle.
Delta m^2 : relation spectrale qui stabilise un rythme de dephasage.
```

Il faut distinguer :

```text
orientation structurelle
manifestation dynamique
protocole d'acces
```

#### 9.6 Convention

Convention introduit une temporalite institutionnelle.

Exemples :

```text
k_B, N_A, c, h, e : valeurs exactes dans le SI revise.
R, K_J, R_K : exactes par composition.
```

Mais cette temporalite ne doit pas etre confondue avec la temporalite physique.

```text
fixer une valeur numerique n'explique pas l'origine physique de la grandeur.
```

### 10. Effet sur les fonctions transversales et architecturales

La temporalite renforce le statut des fonctions transversales.

| Fonction | Lecture temporelle |
|---|---|
| Seuil | Point, zone ou role de passage |
| Borne | Etat d'acces sans detection positive |
| Validite | Domaine ou une description reste applicable |
| Dephasage | Manifestation dynamique d'une relation spectrale |
| Hierarchie | Ordre d'echelles ou d'intensites entre niveaux |

Elle renforce aussi les fonctions architecturales cosmologiques.

| Fonction architecturale | Lecture temporelle |
|---|---|
| Fond | Condition globale de dynamique |
| Parametre d'etat | Etat d'un systeme a une epoque ou dans un modele |
| Densite normalisee | Repartition relative dependante du cadre cosmologique |
| Relation d'etat | Comportement d'une composante, potentiellement dynamique |
| Condition initiale | Donnee primitive d'un scenario d'evolution |
| Observable reconstruite | Etat tardif reconstruit par modeles et acces |

### 11. Effet sur les architectures inter-familles

Les quatre architectures doivent recevoir un profil temporel.

| Architecture | Forme actuelle | Profil temporel propose |
|---|---|---|
| Saveur-Higgs | Constitutive | Temporalite de constitution : brisure, diagonalisation, spectres et orientations deviennent lisibles dans le regime electrofaible brise |
| Metrologique SI | Definitionnelle | Temporalite conventionnelle : une decision metrologique fixe des valeurs numeriques, tandis que les roles physiques restent distincts |
| Effective basse energie | Validite | Temporalite de transition : domaines, echelles, flot, seuils et limites de description |
| Cosmologique | Inferentielle / reconstructive | Temporalite cosmique et epistemique : histoire de l'univers, conditions initiales, etats, observables tardives, routes d'inference |

Cette table ne remplace pas la synthese inter-familles.

Elle ajoute le profil dynamique de chaque architecture.

### 12. Consequence theorique

La proposition theorique devient plus precise.

Formulation anterieure :

```text
constance = robustesse fonctionnelle situee
```

Formulation enrichie :

```text
constance effective stabilisee =
robustesse fonctionnelle situee
dans un regime,
selon un acces,
avec un mode de stabilisation,
et un profil temporel explicite.
```

Cette formulation evite deux erreurs opposees.

Erreur 1 :

```text
penser une constante comme un nombre intemporel isole.
```

Erreur 2 :

```text
transformer toute constante en grandeur fluide ou variable.
```

La bonne position est intermediaire :

```text
une constante peut etre le plateau, le point fixe, la relation robuste,
la convention exacte ou la reconstruction stable d'un processus.
```

### 17. Effet sur la theorie du projet

La reorganisation temporelle rend le projet plus theorique, mais aussi plus prudent.

Elle permet de dire :

```text
une constante effective stabilisee est souvent un noeud stable dans une dynamique,
non un objet immobile hors de tout regime.
```

Mais elle interdit de dire :

```text
tout varie donc toute constance est illusoire.
```

Le projet ne devient pas une theorie generale de la variation des constantes.

Il devient :

```text
une methode pour decrire les formes situees de stabilite dans des dynamiques controlees.
```

### 18. Risques de la revision temporelle

La revision doit eviter cinq risques.

#### 18.1 Inflation temporelle

Erreur :

```text
ajouter un commentaire temporel vague a chaque fiche.
```

Correction :

```text
nommer seulement les temporalites pertinentes.
```

#### 18.2 Confusion entre echelle et temps

Erreur :

```text
traiter Q, E, z, t et date de convention comme la meme variable.
```

Correction :

```text
specifier toujours la variable dynamique.
```

#### 18.3 Speculation cosmologique prematuree

Erreur :

```text
transformer toute temporalite cosmologique en conclusion sur une nouvelle physique.
```

Correction :

```text
rester au niveau regime, acces, modele, tension, limite.
```

#### 18.4 Dissolution de la taxonomie

Erreur :

```text
remplacer les familles par des processus.
```

Correction :

```text
garder les familles comme grammaire fonctionnelle,
ajouter la temporalite comme grammaire dynamique.
```

#### 18.5 Confusion entre histoire du corpus et histoire physique

Erreur :

```text
prendre la genealogie documentaire pour une preuve physique.
```

Correction :

```text
distinguer maturation methodologique et temporalite physique.
```
