# SECTION_C_CONTROLE_PHASE4.md

Statut : registre de contrôle, non destiné à devenir une section des définitions v13.

Objet : conserver la traçabilité de la section C sans importer dans le texte principal les formulations négatives ou les comparaisons avec l'ancien corpus.

## Fonction de la section C

La section C doit :

- proposer une matrice multidimensionnelle d'analyse ;
- organiser les cas sans les réduire à un seul gradient ;
- distinguer domaine, mode d'opération, explicitation, effet, temporalité, dépendance et capacité de retour ;
- préparer les sections D à H.

## Décisions applicables

- D-004 : distinction obligatoire entre les termes de base ;
- D-007 : remplacement de la taxonomie par une matrice multidimensionnelle ;
- D-009 : statut relatif du pré-thématique et des formes d'explicitation ;
- D-010 : réflexivité pluralisée ;
- D-012 : transversalité candidate ;
- D-013 : modalité explicite des propositions ;
- D-015 : distinction des temporalités ;
- D-031 : architecture v13 ;
- D-035 à D-037 : K provisoire, statuts et critères d'échec ;
- D-040 : rédaction positive de la v13 ;
- D-041 : vocabulaire de base de B.

## Micro-architecture

1. fonction de la matrice ;
2. domaine ou site principal ;
3. mode d'opération ;
4. statut d'explicitation ;
5. effet principal ;
6. temporalité ;
7. structure de dépendance ;
8. capacités de retour ;
9. profil de cas ;
10. rôle de la matrice dans les sections suivantes.

## Audit de régression

Contrôles satisfaits :

- la section C est formulée positivement ;
- la matrice ne classe pas des types substantiels ;
- les dimensions sont croisables ;
- le statut d'explicitation est relatif à un agent, collectif, pratique et moment ;
- les capacités de retour ne sont pas présentées comme un gradient unique ;
- la section prépare D, E, F, G et H sans les rédiger ;
- la transversalité reste dépendante des comparaisons futures.

Risques restants :

- la matrice peut devenir trop lourde si chaque cas doit remplir tous les axes au même degré ;
- la notion de "profil de cas" devra rester souple ;
- les profils de résolution sont seulement annoncés et devront être précisés en F ;
- les critères de gain et d'échec restent à stabiliser en H.

Conclusion :

La section C fournit un outil de description et de comparaison. Elle ne clôt pas l'architecture ; elle rend possible la réécriture contrôlée des relations, temporalités, stabilités et devenirs épistémiques.
