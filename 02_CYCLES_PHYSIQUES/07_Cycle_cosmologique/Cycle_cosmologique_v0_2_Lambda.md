# Cycle cosmologique v0.2

## Fiche 1 : Lambda

### 1. Fonction de la fiche

Cette fiche ouvre le traitement des grandeurs cosmologiques proprement dites.

Elle porte sur :

```text
Lambda
```

mais elle ne doit pas reintroduire :

```text
Fond
```

comme famille superieure.

La question directrice est :

> Comment Lambda peut-elle etre fortement stabilisee comme parametre du modele LambdaCDM tout en gardant un statut physique profond ouvert ?

Objectif :

```text
classer Lambda comme terme de fond dans l'architecture cosmologique, sans transformer la cosmologie en famille et sans assimiler Lambda, Omega_Lambda et w.
```

Lecture actuelle apres stress test :

```text
Lambda appartient au sous-reseau fond / expansion.
```

Elle y joue le role de terme de fond.

Elle ne doit pas etre confondue avec :

```text
H_0 comme parametre d'etat actuel,
Omega_Lambda comme densite normalisee,
w comme relation d'etat.
```

### 2. Identification

Grandeur :

```text
Lambda
```

Nom :

```text
constante cosmologique
```

Type logique :

```text
terme dans les equations gravitationnelles
parametre du modele LambdaCDM
source effective d'acceleration cosmique
```

Dimension :

```text
[Lambda] = L^-2
```

Statut de rang :

```text
fonction architecturale principale : Fond
architecture : Cosmologique
sous-reseau : Fond / expansion
famille superieure : aucune famille simple ne doit etre forcee
```

Lectures secondaires :

```text
Echelle, si l'on lit Lambda par une longueur caracteristique Lambda^(-1/2)
Relation, si l'on lit Omega_Lambda comme densite normalisee
Relation d'etat, si l'on lit w = -1 pour une composante Lambda stricte
```

Mais ces lectures secondaires ne remplacent pas le statut principal :

```text
Lambda = terme de fond dans l'architecture cosmologique
```

### 3. Pourquoi Lambda est un cas dangereux

Lambda est un cas dangereux pour la taxonomie parce qu'il attire plusieurs confusions.

Premiere confusion :

```text
Lambda = famille Fond
```

Correction :

```text
Fond est une fonction architecturale, non une famille superieure.
```

Deuxieme confusion :

```text
Lambda = toute l'energie noire
```

Correction :

```text
Lambda est une forme particuliere d'energie noire : densite constante, equation d'etat w = -1. Les modeles dynamiques d'energie noire ne sont pas identiques a Lambda.
```

Troisieme confusion :

```text
Lambda = Omega_Lambda
```

Correction :

```text
Lambda est un terme dimensionne ; Omega_Lambda est une densite normalisee dependante du modele et de l'epoque de reference.
```

Quatrieme confusion :

```text
Lambda fortement ajuste = Lambda physiquement explique
```

Correction :

```text
la stabilisation empirique dans LambdaCDM ne resout pas le statut physique profond de Lambda.
```

### 4. Regime de definition physique

Dans les equations d'Einstein, Lambda intervient sous la forme :

```text
G_mu_nu + Lambda g_mu_nu = (8 pi G / c^4) T_mu_nu
```

Elle peut etre deplacee du cote du tenseur energie-impulsion et lue comme une composante effective de fond.

Dans une lecture de fluide cosmologique, elle correspond a une densite d'energie uniforme et a une pression negative avec :

```text
w = p / rho = -1
```

Dans le modele LambdaCDM, Lambda joue donc le role d'un terme responsable de l'acceleration tardive de l'expansion cosmique.

Le regime de definition physique est :

```text
dynamique globale de l'espace-temps dans une cosmologie relativiste
```

et non :

```text
interaction locale
couplage de particules
constante definissante d'un systeme d'unites
```

### 5. Regime d'acces epistemique

Lambda n'est pas mesuree comme une constante isolee.

Elle est inferee dans un modele cosmologique a partir d'un reseau de sondes :

```text
CMB
BAO
supernovae de type Ia
lentille gravitationnelle
croissance des structures
ajustements combines
```

Le regime d'acces est donc :

```text
inference cosmologique modele-dependante
```

Il faut distinguer :

```text
regime physique : role de Lambda dans la dynamique cosmologique
regime d'acces : ajustements de donnees sous hypotheses de modele
```

Cette distinction est centrale. Une meme grandeur peut etre fortement stabilisee dans un modele et rester ouverte dans son interpretation physique.

### 6. Lambda, Omega_Lambda et rho_Lambda

Lambda ne doit pas etre confondue avec ses reformulations.

On peut associer a Lambda une densite effective d'energie noire :

```text
rho_Lambda = Lambda c^2 / (8 pi G)
```

selon les conventions de densite utilisees.

On peut aussi definir une densite normalisee :

```text
Omega_Lambda = rho_Lambda / rho_crit
```

avec :

```text
rho_crit = 3 H_0^2 / (8 pi G)
```

La distinction taxonomique est :

```text
Lambda        : terme de fond dimensionne
rho_Lambda    : densite effective associee
Omega_Lambda  : densite normalisee, relationnelle et modele-dependante
```

Ainsi :

```text
Omega_Lambda
```

releve plutot d'une :

```text
rapport normalise
```

et d'une fonction architecturale :

```text
Densite normalisee
```

La presente fiche porte sur Lambda, non sur Omega_Lambda.

### 7. Lambda et w

Pour une constante cosmologique stricte :

```text
w = -1
```

Cela signifie :

```text
p = - rho
```

Mais w n'est pas Lambda.

Taxonomiquement :

```text
Lambda : terme de fond
w      : relation d'etat
```

Dans les modeles d'energie noire dynamique, on peut utiliser :

```text
w(a) = w_0 + w_a(1-a)
```

ou d'autres parametrisations.

Ces parametrisations testent des deviations possibles par rapport a Lambda, mais elles ne doivent pas etre confondues avec une explication physique de l'energie noire.

Formulation prudente :

```text
w = -1 est la signature relationnelle d'une composante Lambda stricte.
w(a) variable est une parametrisation phenomenologique d'une energie noire dynamique possible.
```

### 8. Stabilisation empirique

Planck 2018 donne une tres forte stabilisation du modele de base LambdaCDM dans le CMB.

Dans le cadre plat a six parametres, Planck trouve notamment une bonne coherence globale du modele, avec :

```text
H_0 = 67.4 +/- 0.5 km/s/Mpc
Omega_m = 0.315 +/- 0.007
sigma_8 = 0.811 +/- 0.006
```

Ces valeurs ne sont pas des mesures directes de Lambda isolee. Elles sont des parametres inferes dans un modele.

Planck 2018 ne donne pas, dans son cadre principal, de preuve convaincante en faveur d'extensions au modele de base LambdaCDM.

Mais le statut actuel doit integrer DESI DR2.

DESI DR2, publie en mars 2025, presente des mesures BAO sur trois ans de donnees. Les resultats BAO restent bien decrits par un modele plat LambdaCDM, mais les combinaisons avec CMB et supernovae favorisent des modeles d'energie noire dynamique de type w0waCDM, avec des significativites dependantes des jeux de donnees.

La formulation prudente est :

```text
LambdaCDM reste un modele de reference fortement stabilise.
Mais certaines combinaisons recentes de donnees, notamment DESI DR2 + CMB + supernovae, exercent une pression notable en faveur de modeles d'energie noire dynamique.
```

Il ne faut donc pas ecrire :

```text
LambdaCDM est abandonne.
```

Il faut ecrire :

```text
LambdaCDM est teste de facon plus severe par les donnees recentes.
```

### 9. Statut DESI au 9 juillet 2026

Au 9 juillet 2026, le point de prudence est le suivant.

DESI DR2 correspond aux analyses publiees en mars 2025 sur les trois premieres annees de donnees.

DESI a annonce le 15 avril 2026 l'achevement de son releve planifie de cinq ans, avec plus de 47 millions de galaxies et quasars observes. Cela ne signifie pas encore que toutes les contraintes cosmologiques finales du releve complet sont publiees.

La bonne formulation est donc :

```text
les resultats DESI DR2 constituent une contrainte actuelle majeure ;
les analyses completes du releve planifie de cinq ans restent a integrer lorsqu'elles seront publiees.
```

Cette precision evite de confondre :

```text
fin d'observation
publication d'analyse cosmologique
revision stabilisee de la taxonomie
```

### 10. Stabilisation conventionnelle

Lambda n'est pas une constante definissante du SI.

Sa valeur n'est pas fixee par convention metrologique.

Stabilisation conventionnelle :

```text
non concernee
```

Lambda peut etre exprimee dans des unites differentes, mais son statut physique ne vient pas d'une definition d'unite.

### 12. Ce que Lambda confirme

Lambda confirme la necessite d'une architecture cosmologique.

Elle ne se laisse pas bien classer comme :

```text
Couplage
Echelle
Relation
Raccordement
Orientation
Convention
```

sans perte.

Elle peut avoir des lectures secondaires par echelle, densite ou relation d'etat, mais sa fonction principale dans le cycle est :

```text
terme de fond dans une architecture globale
```

La carte doit donc garder un niveau :

```text
Fonction architecturale
```

distinct du niveau :

```text
Famille superieure
```

### 13. Ce que Lambda ne confirme pas

Lambda ne confirme pas :

```text
Fond comme famille superieure
```

Elle ne confirme pas non plus :

```text
Cosmologie comme famille
```

Elle ne resout pas :

```text
la nature physique de l'energie noire
le probleme de la constante cosmologique
la tension entre inference CMB et mesures locales de H_0
les tensions de croissance des structures
```

Elle fournit plutot un cas central ou la stabilisation empirique, la puissance phenomenologique et l'ouverture interpretative coexistent.

### 14. Decision locale

Decision de classement :

```text
Lambda = terme de fond dans l'architecture cosmologique.
```

Statut :

```text
fonction architecturale : Fond
architecture : Cosmologique
sous-reseau : Fond / expansion
stabilisation empirique : forte dans LambdaCDM, mais actuellement testee par DESI DR2 et autres combinaisons de donnees
stabilisation conventionnelle : non concernee
```

Formulation finale :

> Lambda est un terme de fond dans l'architecture cosmologique ; sa valeur est fortement stabilisee comme parametre du modele LambdaCDM, mais son statut physique profond et son caractere strictement constant restent ouverts, surtout dans le contexte des tests recents d'energie noire dynamique.

### 15. Effet sur le cycle cosmologique

La fiche Lambda fixe le premier pilier du cycle.

Elle impose trois regles pour la suite.

Premiere regle :

```text
ne pas confondre terme, densite normalisee et relation d'etat.
```

Deuxieme regle :

```text
ne pas confondre stabilisation dans un modele et explication physique.
```

Troisieme regle :

```text
traiter les tensions comme proprietes du regime d'acces, non comme categories taxonomiques.
```

Lecture apres stress test :

```text
Lambda et H_0 forment le noyau fond / expansion ;
Lambda stabilise le terme de fond,
H_0 stabilise l'etat actuel de l'expansion.
```
