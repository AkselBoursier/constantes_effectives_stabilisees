# Cycle cosmologique v0.5

## Fiche 4 : w

### 1. Fonction de la fiche

Cette fiche porte sur :

```text
w
```

le parametre d'equation d'etat d'une composante cosmologique.

La question directrice est :

> Comment w peut-il etre une relation d'etat et une parametrisation phenomenologique sans devenir une explication physique de l'energie noire ?

Objectif :

```text
classer w comme relation d'etat dans l'architecture cosmologique, en distinguant w = -1, w constant et w(a) dynamique.
```

La fiche doit eviter quatre erreurs.

Premiere erreur :

```text
w = Lambda
```

Deuxieme erreur :

```text
w = explication physique de l'energie noire
```

Troisieme erreur :

```text
w0waCDM = preuve directe d'une nouvelle physique
```

Quatrieme erreur :

```text
tension DESI sur w = famille taxonomique
```

La bonne formulation est :

```text
w est une relation d'etat ou une parametrisation phenomenologique du comportement d'une composante cosmologique, connue par inference modele-dependante.
```

### 2. Identification

Grandeur :

```text
w
```

Nom :

```text
parametre d'equation d'etat
```

Definition usuelle :

```text
w = p / rho
```

ou, selon les conventions d'unites :

```text
w = p / (rho c^2)
```

Type logique :

```text
relation d'etat
```

Dimension :

```text
sans dimension
```

Statut de rang :

```text
famille principale : Relation
sous-type : relation d'etat
fonction architecturale : Relation d'etat
architecture : Cosmologique
```

### 3. Pourquoi w n'est pas Lambda

La fiche Lambda a pose :

```text
Lambda = terme de fond
```

La fiche Omega_i a pose :

```text
Omega_Lambda = densite normalisee associee a la composante Lambda
```

La presente fiche pose :

```text
w = relation d'etat
```

Ces trois objets sont solidaires, mais non identiques.

Pour une constante cosmologique stricte :

```text
w = -1
```

Mais cette valeur ne signifie pas :

```text
w est Lambda
```

Elle signifie :

```text
la composante Lambda stricte a une equation d'etat p = -rho
```

dans les conventions ou c = 1.

Distinction taxonomique :

```text
Lambda        : terme de fond dimensionne
rho_Lambda    : densite associee
Omega_Lambda  : densite normalisee
w             : relation d'etat
```

### 4. Regime de definition physique

Le regime de definition physique est celui d'une description cosmologique par composantes fluides.

Une composante est caracterisee par :

```text
densite d'energie
pression
equation d'etat
```

La relation :

```text
p = w rho
```

ou :

```text
p = w rho c^2
```

selon les conventions, decrit le comportement de la composante dans la dynamique d'expansion.

Exemples standards :

```text
matiere non relativiste : w = 0
rayonnement             : w = 1/3
constante cosmologique  : w = -1
```

Dans le cycle present, l'objet principal est :

```text
w de l'energie noire
```

ou plus prudemment :

```text
relation d'etat de la composante responsable de l'acceleration tardive
```

### 5. Regime d'acces epistemique

w n'est pas mesure comme une grandeur locale directe.

Il est infere par l'effet de l'energie noire sur :

```text
histoire d'expansion
distances cosmologiques
BAO
supernovae
CMB
lentille gravitationnelle
croissance des structures
ajustements combines
```

Le regime d'acces est donc :

```text
inference cosmologique modele-dependante
```

Il faut distinguer :

```text
regime physique : relation pression / densite d'une composante
regime d'acces : contraintes sur l'expansion et les distances sous hypotheses de modele
```

Cette distinction est centrale, parce qu'une contrainte sur w n'est pas directement une identification de la nature microscopique de l'energie noire.

### 6. Trois lectures de w

La fiche distingue trois niveaux.

#### 6.1 LambdaCDM

Dans LambdaCDM :

```text
w = -1
```

La valeur n'est pas un parametre libre du modele de base.

Elle exprime la relation d'etat d'une constante cosmologique stricte.

Lecture :

```text
w = -1 comme signature relationnelle de Lambda
```

#### 6.2 wCDM

Dans wCDM :

```text
w = constante libre
```

Le modele teste une deviation constante par rapport a :

```text
w = -1
```

Lecture :

```text
w constant comme extension phenomenologique minimale
```

Ce test ne donne pas encore une dynamique temporelle.

#### 6.3 w0waCDM

Dans w0waCDM, on utilise souvent :

```text
w(a) = w_0 + w_a(1-a)
```

ou, en fonction du redshift :

```text
w(z) = w_0 + w_a z / (1+z)
```

Lecture :

```text
w_0 : valeur actuelle
w_a : variation effective avec l'echelle cosmique
```

Ce modele donne une parametrisation de l'evolution possible de l'energie noire.

Il ne donne pas, par lui-meme :

```text
le champ responsable
le potentiel
la theorie de gravitation modifiee
la nature microscopique de l'energie noire
```

### 7. w comme relation d'etat

Le classement principal propose est :

```text
famille : Relation
sous-type : relation d'etat
```

Cette classification est plus precise que :

```text
w = parametre cosmologique
```

Le mot "parametre" dit comment w est utilise dans un modele.

La formule :

```text
w = p / rho
```

dit sa fonction.

La fonction principale est donc :

```text
stabiliser une relation entre pression et densite
```

Cette fonction confirme la regle v1.1 :

```text
Relation doit toujours etre qualifiee.
```

### 8. w comme fonction architecturale

w joue aussi une fonction architecturale.

Dans l'architecture cosmologique, il indique comment une composante contribue a la dynamique d'expansion.

Avec l'equation de conservation, pour un w constant :

```text
rho(a) ~ a^(-3(1+w))
```

Donc :

```text
w = 0     -> rho_m ~ a^-3
w = 1/3   -> rho_r ~ a^-4
w = -1    -> rho_Lambda = constante
```

Cette fonction n'est pas une famille nouvelle.

Elle est une fonction architecturale :

```text
Relation d'etat
```

### 9. w et les donnees recentes

Planck 2018 stabilise fortement le modele de base LambdaCDM dans son cadre principal et ne trouve pas, dans ce cadre, de preuve convaincante en faveur d'extensions au modele de base.

Le statut actuel doit toutefois integrer DESI DR2.

DESI DR2, dans son analyse BAO sur trois ans de donnees, indique que les resultats sont bien decrits par un modele plat LambdaCDM, mais que certaines combinaisons avec CMB et supernovae preferent une equation d'etat dynamique parametree par :

```text
w_0
w_a
```

La preference DESI DR2 + CMB pour w0waCDM est donnee autour de :

```text
3.1 sigma
```

et, avec supernovae, varie selon les echantillons :

```text
2.8 a 4.2 sigma
```

La note DESI DR2 et l'analyse etendue indiquent que l'indice d'une energie noire dynamique, notamment a bas redshift, est robuste sous plusieurs choix de modelisation.

Mais la formulation prudente reste :

```text
les donnees recentes exercent une pression forte sur LambdaCDM ;
elles ne constituent pas encore une explication physique definitive de l'energie noire.
```

Il ne faut pas ecrire :

```text
DESI a prouve que Lambda est fausse.
```

Il faut ecrire :

```text
DESI DR2 motive un test serieux des parametrisations dynamiques de w.
```

### 10. Phantom crossing et prudence de statut

Les analyses DESI etendues discutent une preference pour des modeles avec passage autour de :

```text
w = -1
```

Ce point est souvent appele :

```text
phantom crossing
```

Il doit etre traite prudemment.

Taxonomiquement, ce n'est pas :

```text
une famille
une constante
une conclusion physique directe
```

C'est :

```text
une propriete d'une parametrisation ou d'une classe de modeles ajustee aux donnees.
```

Regle :

```text
phantom crossing appartient au regime d'inference et de modelisation, non a la famille principale de w.
```

### 11. Stabilisation empirique

La stabilisation empirique de w est ambivalente.

Elle est forte pour :

```text
w = -1 dans LambdaCDM comme modele de reference
```

au sens ou LambdaCDM reste un cadre tres contraint et predictif.

Elle devient ouverte pour :

```text
w constant libre
w(a) dynamique
```

car les resultats dependent fortement des combinaisons de donnees et des parametrisations.

Lecture :

| Regime | Statut de w |
|---|---|
| LambdaCDM | w fixe a -1 |
| wCDM | w constant libre, contrainte phenomenologique |
| w0waCDM | w(a) dynamique, contrainte par combinaisons de donnees |
| Reconstructions non parametriques | test de robustesse, plus grande flexibilite |

Formulation correcte :

```text
w est fortement structure par les donnees, mais sa lecture dynamique reste modele-dependante.
```

### 12. Stabilisation conventionnelle

w n'est pas une constante definissante du SI.

Sa valeur n'est pas fixee par convention metrologique.

Stabilisation conventionnelle :

```text
non concernee
```

w est sans dimension, mais cela ne le rend pas conventionnellement exact.

### 14. Ce que w confirme

w confirme que Relation reste necessaire comme famille.

Mais il confirme aussi que Relation ne doit jamais rester non qualifiee.

La bonne classification n'est pas :

```text
w = Relation
```

Elle est :

```text
w = relation d'etat
```

La fonction architecturale est :

```text
Relation d'etat
```

Ce cas confirme aussi la separation entre :

```text
terme de fond
densite normalisee
relation d'etat
```

### 15. Ce que w ne confirme pas

w ne confirme pas :

```text
Cosmologie comme famille
```

Il ne confirme pas :

```text
Fond comme famille
```

Il ne confirme pas non plus :

```text
w0waCDM comme explication physique complete de l'energie noire
```

Il ne faut pas confondre :

```text
parametrisation qui ajuste mieux certaines donnees
theorie physique expliquant la cause de l'acceleration cosmique
```

### 16. Decision locale

Decision de classement :

```text
w = relation d'etat dans l'architecture cosmologique.
```

Statut :

```text
famille principale : Relation
sous-type : relation d'etat
fonction architecturale : Relation d'etat
architecture : Cosmologique
stabilisation empirique : forte pour la reference w = -1, sous pression dans les tests dynamiques recents
stabilisation conventionnelle : non concernee
```

Formulation finale :

> w n'est pas Lambda et n'est pas une explication physique de l'energie noire ; c'est la relation d'etat qui relie pression et densite d'une composante cosmologique, et dont les parametrisations dynamiques servent a tester les deviations possibles par rapport a LambdaCDM.

### 17. Effet sur le cycle cosmologique

Apres Lambda, H_0, Omega_i et w, le cycle distingue quatre niveaux.

```text
Lambda  : terme de fond
H_0     : parametre d'etat actuel
Omega_i : densite normalisee / relation normalisee
w       : relation d'etat
```

Ces niveaux sont solidaires.

```text
Lambda stricte implique w = -1
Omega_Lambda normalise la densite associee a Lambda
w(a) teste une composante dynamique possible
H_0 et les distances contraignent indirectement w
```

Mais ces niveaux ne doivent pas etre fusionnes.

### 19. Effet sur la carte et la methode

La fiche w renforce trois regles methodologiques.

Premiere regle :

```text
une relation d'etat est une relation, pas un fond.
```

Deuxieme regle :

```text
une parametrisation phenomenologique n'est pas une explication physique.
```

Troisieme regle :

```text
une preference de donnees appartient d'abord au regime d'acces et de modelisation.
```

La fiche ajoute aussi une precaution pour la suite :

```text
les conditions initiales perturbatives ne devront pas etre classees comme Fond par simple appartenance cosmologique.
```
