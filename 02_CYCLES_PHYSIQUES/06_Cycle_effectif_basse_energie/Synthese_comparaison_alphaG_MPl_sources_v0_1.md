### 3. Difference utile

La source A porte une formulation plus directe.

Elle insiste sur :

```text
G comme grandeur dimensionnee ;
la necessite de passer par un couplage sans dimension ;
M_Pl comme echelle de comparaison ;
la categorie ancienne de hierarchie d'echelle.
```

La source B reformule et articule davantage.

Elle insiste sur :

```text
la gravitation basse energie comme theorie effective ;
la suppression du couplage quantique gravitationnel ;
le caractere multi-echelle de la gravitation ;
la transition vers les rapports sans dimension pour toute variation.
```

La fiche active v0.4 a deja depasse les deux sources.

Elle ajoute notamment :

```text
la difference entre M_Pl et masse de Planck reduite ;
la comparaison avec G_F / M_W ;
la comparaison avec alpha_s(Q^2) / Lambda_QCD ;
la fonction transversale de validite ;
la decision de ne pas creer une nouvelle famille de hierarchie.
```

### 4. Recuperation conceptuelle

Le point recupere pour le cercle 2 est clair.

```text
alpha_G(E) / M_Pl ne fournit pas une famille nouvelle ;
il stabilise une architecture effective basse energie par suppression
d'echelle et par limite de validite attendue.
```

Le cas gravitationnel force trois distinctions.

```text
1. G_N est dimensionnee ; le contenu physique robuste passe par des rapports.
2. M_Pl est une echelle de reference, pas une frontiere experimentale nette.
3. La basse energie gravitationnelle est un regime de suppression, non le
   meme type de basse energie que QCD.
```

### 5. Effet sur l'architecture effective

L'architecture effective basse energie est renforcee.

Elle ne se reduit pas a un seul schema.

```text
G_F / M_W                  -> approximation locale sans boson W explicite
alpha_s(Q^2) / Lambda_QCD -> flot et changement de regime
alpha_G(E) / M_Pl         -> suppression par hierarchie d'echelle
```

Pour le cercle 2, la question devient :

```text
comment decrire une architecture effective pluralisee sans multiplier les
categories locales inutiles ?
```

### 6. Decision de recuperation

Decision :

```text
le doublon alpha_G / M_Pl est traite comme genealogie a deux etats ;
B ne remplace pas A, et A ne doit pas effacer B.
```

Pour l'usage courant :

```text
la fiche active v0.4 reste la reference de travail ;
les sources A et B servent de controle genealogique.
```
