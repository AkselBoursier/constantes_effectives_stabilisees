# Synthese du cycle effectif basse energie v0.1

## Architecture effective basse energie, validite et formes d'effectivite

### 1. Fonction de cette synthese

Le cycle effectif basse energie avait une fonction precise : tester si l'architecture effective basse energie est une architecture reelle, ou seulement un regroupement commode de cas deja classables ailleurs.

La question initiale etait :

> Qu'est-ce qui devient constant lorsqu'une description plus complete est remplacee, dans un domaine limite, par une description effective ?

Le cycle a porte sur quatre cas :

```text
G_F
M_W
alpha_s(Q^2) / Lambda_QCD
alpha_G(E) / M_Pl
```

Ces cas ne sont pas equivalents. C'est leur difference qui fait la force du cycle.

Le premier cas teste un coefficient de raccordement effectif. Le deuxieme teste une echelle de validite. Le troisieme teste un couplage courant et une echelle dynamique. Le quatrieme teste un couplage gravitationnel dimensionne reformule par rapport a une echelle de validite attendue.

L'objectif de cette synthese est donc double :

```text
1. stabiliser ce que le cycle confirme ;
2. fixer les limites de l'architecture pour eviter que "effectif" devienne une famille trop large.
```

### 2. Resultat general

Le cycle confirme l'architecture effective basse energie comme architecture inter-familles.

Il ne confirme pas une nouvelle famille superieure.

La decision est :

```text
Architecture effective basse energie = architecture robuste, mais non famille.
```

Elle articule plusieurs familles deja stabilisees :

```text
Raccordement
Echelle
Couplage
Relation, lorsque des liaisons de regime sont explicites
```

Elle mobilise aussi plusieurs fonctions transversales :

```text
Validite
Seuil
Hierarchie
```

Mise a jour v1.3 :

```text
le cycle doit maintenant etre lu avec le rang pre-familial,
Relation comme famille faible sous audit,
et la trajectoire de stabilisation des regimes effectifs.
```

La formule provisoirement stabilisee devient :

> Une architecture effective basse energie stabilise les grandeurs qui rendent un regime descriptible sans resoudre explicitement tout ce qui appartient a un niveau, une echelle ou une dynamique plus profonde.

Cette formule doit rester methodologique. Elle ne dit pas qu'une theorie effective serait moins reelle. Elle dit qu'une description est adaptee a un domaine de validite, avec ses propres degres de liberte, coefficients, echelles et limites.

### 3. Les quatre cas du cycle

#### 3.1 G_F : raccordement effectif local

La constante de Fermi :

```text
G_F
```

est le cas le plus net du cycle.

Sa fonction principale est :

```text
Raccordement
```

Sous-type :

```text
raccordement effectif basse energie
```

Dans le regime :

```text
E << M_W
```

l'echange du boson W n'est pas resolu comme propagateur explicite. Son effet est condense dans une interaction locale a quatre fermions, parametree par G_F.

Le point methodologique est decisif :

```text
G_F n'est pas seulement une valeur tres precise.
G_F est le coefficient qui rend robuste une description effective.
```

La relation avec le secteur electrofaible complet existe, notamment avec :

```text
g
M_W
v
```

mais la famille principale de G_F n'est pas Relation. La relation est presente ; la fonction principale est le raccordement entre deux descriptions.

Decision locale :

```text
G_F = Raccordement effectif basse energie
```

#### 3.2 M_W : echelle de validite de la description de Fermi

La masse du boson W :

```text
M_W
```

est d'abord une echelle du regime electrofaible brise.

Sa famille principale est :

```text
Echelle
```

Sous-type :

```text
echelle electrofaible de jauge
```

Mais M_W joue aussi une fonction transversale :

```text
Seuil / Validite
```

Cette fonction apparait lorsqu'on lit M_W depuis la theorie de Fermi. A basse energie, l'interaction faible peut etre traitee comme locale. A mesure que l'on approche l'echelle du W, cette description cesse d'etre suffisante et la theorie electrofaible complete doit etre reintroduite.

La distinction importante est :

```text
M_W comme famille principale : Echelle
M_W comme fonction transversale : seuil de validite
```

Ce cas confirme la retrogradation de Seuil :

```text
Seuil n'est pas une famille superieure.
Seuil est une fonction que peut jouer une echelle.
```

Decision locale :

```text
M_W = Echelle electrofaible a fonction de validite
```

#### 3.3 alpha_s(Q^2) / Lambda_QCD : flot de couplage et changement de regime

Le cas QCD introduit une structure differente.

Les grandeurs principales sont :

```text
alpha_s(Q^2)
Lambda_QCD
```

Leur classement est :

```text
alpha_s(Q^2) : Couplage
Lambda_QCD   : Echelle
```

Sous-types :

```text
alpha_s(Q^2) : couplage courant
Lambda_QCD   : echelle dynamique
```

Ce cas corrige une vision trop simple de l'effectivite.

Dans le cas faible :

```text
basse energie = simplification locale
```

Dans le cas QCD :

```text
basse energie = entree dans le non perturbatif
```

La basse energie QCD n'est donc pas une simplification au sens de la theorie de Fermi. Elle correspond a un changement de regime : le couplage fort grandit, la description perturbative cesse d'etre adaptee, et les degres de liberte hadroniques deviennent pertinents.

Le raccordement n'est pas ici porte par une constante unique comparable a G_F. Il est architectural :

```text
couplage courant
+ echelle dynamique
+ changement de regime
+ modification des degres de liberte
```

Decision locale :

```text
alpha_s(Q^2) / Lambda_QCD = effectivite par flot de couplage et changement de regime
```

#### 3.4 alpha_G(E) / M_Pl : couplage dimensionne et suppression d'echelle

Le cas gravitationnel introduit une troisieme forme.

Les grandeurs principales sont :

```text
G_N
alpha_G(E)
M_Pl
```

Le classement stabilise est :

```text
alpha_G(E) : Couplage
M_Pl       : Echelle
G_N        : constante dimensionnee de depart, a lire par ses rapports et ses effets
```

Le point crucial est que G_N ne doit pas etre traite comme un nombre directement comparable a alpha. G_N est dimensionnee. Son contenu de couplage devient plus propre lorsqu'on construit une grandeur sans dimension :

```text
alpha_G(E) ~ (E / M_Pl)^2
```

Le regime basse energie gravitationnel est :

```text
E << M_Pl
```

Dans ce regime, le couplage quantique gravitationnel effectif est fortement supprime.

Le cas gravitationnel ne ressemble donc ni au cas faible, ni au cas QCD.

Il ne s'agit pas d'integrer un mediateur massif deja connu :

```text
pas de structure analogue simple a W -> G_F
```

Il ne s'agit pas non plus d'une entree dans le non perturbatif a basse energie :

```text
la basse energie gravitationnelle est un regime de suppression
```

La fonction de M_Pl est :

```text
Echelle de validite attendue
```

avec fonctions transversales :

```text
Validite
Hierarchie
Seuil faible
```

Decision locale :

```text
alpha_G(E) / M_Pl = effectivite par couplage dimensionne et suppression d'echelle
```

### 4. Trois formes d'effectivite stabilisees

Le cycle permet maintenant de distinguer trois formes internes d'effectivite.

| Forme | Cas | Structure |
|---|---|---|
| Integration d'un mediateur massif | G_F / M_W | Un degre de liberte lourd n'est pas resolu ; son effet devient coefficient local |
| Flot de couplage et changement de regime | alpha_s(Q^2) / Lambda_QCD | Le couplage varie avec l'echelle ; une echelle dynamique marque le changement de description |
| Couplage dimensionne et suppression d'echelle | alpha_G(E) / M_Pl | Le couplage effectif depend d'un rapport d'echelle ; la basse energie est un domaine de suppression |

Cette distinction est le resultat principal du cycle.

Elle empeche de reduire l'effectivite a une seule image :

```text
effectif = integrer une particule lourde
```

Cette image est correcte pour G_F / M_W, mais elle ne couvre ni QCD, ni la gravitation.

Formulation consolidee :

> L'effectivite basse energie peut etre locale, dynamique ou hierarchique selon ce qui devient non resolu, non perturbatif ou fortement supprime dans le regime considere.

### 5. Familles impliquees

L'architecture effective basse energie ne remplace pas les familles de la methode v1.3.

Elle les articule.

| Grandeur | Famille principale | Sous-type | Fonction transversale |
|---|---|---|---|
| G_F | Raccordement | Raccordement effectif basse energie | Validite, hierarchie, echelle derivee |
| M_W | Echelle | Echelle electrofaible de jauge | Seuil, validite, hierarchie |
| alpha_s(Q^2) | Couplage | Couplage courant | Validite, hierarchie |
| Lambda_QCD | Echelle | Echelle dynamique | Seuil, validite, hierarchie |
| alpha_G(E) | Couplage | Couplage gravitationnel effectif de regime | Validite, hierarchie |
| M_Pl | Echelle | Echelle gravitationnelle de validite attendue | Validite, hierarchie, seuil faible |

Relation reste une famille faible sous audit.

Elle peut intervenir lorsque des liaisons de regime sont explicites, mais elle ne doit pas absorber le cycle.

Elle intervient lorsque l'on explicite les liaisons :

```text
G_F avec g, M_W, v
M_W avec g, v, theta_W
alpha_G(E) avec E / M_Pl
```

Mais ces relations ne suffisent pas a classer les grandeurs. La priorite reste la forme logique, la fonction principale, le regime et la trajectoire :

```text
G_F raccorde
M_W fixe une echelle
alpha_s couple
Lambda_QCD marque une echelle dynamique
alpha_G(E) couple par rapport a une echelle
M_Pl hierarchise une validite attendue
```

### 6. Ce que le cycle stabilise sur "basse energie"

Le cycle montre que "basse energie" n'est pas un concept uniforme.

| Domaine | Sens de basse energie | Effet principal |
|---|---|---|
| Interaction faible | E << M_W | Simplification locale par G_F |
| QCD | Q proche ou inferieur a Lambda_QCD | Entree dans le non perturbatif |
| Gravitation | E << M_Pl | Suppression du couplage quantique gravitationnel |

Il faut donc eviter :

```text
basse energie = toujours plus simple
```

La bonne formulation est :

```text
basse energie = regime ou certains degres de liberte, certaines echelles ou certaines corrections ne sont pas traites de la meme maniere que dans la description plus complete
```

Dans le cas faible, cela simplifie.

Dans le cas QCD, cela rend la description perturbative insuffisante.

Dans le cas gravitationnel, cela rend la gravitation quantique fortement supprimee.

La trajectoire du cycle est donc :

```text
description complete ou plus profonde
-> regime limite
-> non-resolution explicite de certains degres de liberte
-> stabilisation locale d'un coefficient, d'un couplage ou d'une echelle
-> domaine de validite explicite
```

### 7. Statut du raccordement

Le cycle oblige a distinguer deux types de raccordement.

Premier type :

```text
raccordement local porte par une constante
```

Exemple :

```text
G_F
```

Ici, une grandeur porte clairement la fonction de raccordement entre la theorie de Fermi et la theorie electrofaible.

Deuxieme type :

```text
raccordement architectural
```

Exemples :

```text
alpha_s(Q^2) / Lambda_QCD
alpha_G(E) / M_Pl
```

Dans ces cas, aucune constante unique ne joue exactement le role de G_F.

Pour QCD, le passage de regime se distribue entre :

```text
couplage courant
echelle dynamique
degres de liberte adaptes
methodes perturbatives ou non perturbatives
```

Pour la gravitation, le raccordement est encore plus prudent :

```text
relativite generale effective basse energie
completion quantique inconnue ou ouverte
echelle de Planck comme indice de validite attendue
```

Decision :

```text
Raccordement reste une famille lorsqu'une grandeur porte principalement cette fonction.
Mais l'architecture effective basse energie peut contenir des raccordements distribues ou incomplets.
```

### 8. Statut de Seuil

Le cycle confirme la decision v0.8 :

```text
Seuil n'est pas une famille superieure.
```

M_W joue une fonction seuil pour la theorie de Fermi.

Lambda_QCD joue une fonction seuil pour le passage perturbatif / non perturbatif.

M_Pl joue une fonction seuil faible ou attendue pour la validite de la gravitation effective basse energie.

Mais dans les trois cas, le classement principal reste ailleurs :

```text
M_W        : Echelle
Lambda_QCD : Echelle
M_Pl       : Echelle
```

La fonction seuil est donc reelle, mais secondaire.

Formulation :

> Un seuil est souvent la maniere dont une echelle devient lisible depuis une description qui cesse d'etre suffisante.

### 9. Statut de Validite

La fonction Validite sort renforcee du cycle.

Elle est plus precise que Seuil.

Le seuil dit :

```text
ou se situe la limite ou la zone de passage ?
```

La validite demande :

```text
quelle description reste applicable dans quel domaine ?
```

Le cycle montre trois formes de validite :

| Grandeur | Fonction de validite |
|---|---|
| M_W | Validite de la theorie de Fermi comme approximation locale |
| Lambda_QCD | Validite relative des descriptions perturbatives en QCD |
| M_Pl | Validite attendue de la relativite generale comme theorie effective basse energie |

La fonction Validite doit donc rester centrale dans l'architecture effective basse energie.

### 10. Statut de Hierarchie

La fonction Hierarchie est egalement renforcee.

Elle intervient dans les trois formes du cycle :

```text
E << M_W
Q par rapport a Lambda_QCD
E << M_Pl
```

Mais elle ne doit pas devenir une famille autonome.

La hierarchie organise des ecarts d'echelle, de couplage ou de validite. Elle qualifie la lecture d'une grandeur. Elle ne remplace pas sa famille principale.

Decision :

```text
Hierarchie = fonction transversale forte, non famille superieure.
```

Ce point corrige l'ancienne formulation :

```text
constante de hierarchie d'echelle
```

La formulation v0.8 correcte est :

```text
couplage ou echelle a fonction hierarchique
```

### 11. Definition consolidee de l'architecture effective basse energie

Definition proposee :

> Une architecture effective basse energie est une configuration inter-familles dans laquelle un regime limite stabilise des coefficients, couplages, echelles et fonctions de validite permettant de decrire un domaine sans resoudre explicitement toute la structure pertinente a plus haute energie, a plus courte distance ou dans un autre regime de description.

Cette definition contient quatre elements.

Premier element :

```text
regime limite
```

La description vaut dans un domaine determine.

Deuxieme element :

```text
grandeurs stabilisees
```

Certaines grandeurs deviennent robustes dans ce domaine.

Troisieme element :

```text
non-resolution
```

Quelque chose n'est pas resolu explicitement : un mediateur lourd, un regime de couplage fort, des corrections quantiques gravitationnelles, ou une completion inconnue.

Quatrieme element :

```text
validite
```

L'architecture doit toujours dire ou la description est applicable et ce qu'elle ne dit pas.

### 12. Formule compacte

La formule compacte de l'architecture peut etre ecrite ainsi :

```text
Architecture effective basse energie =
Raccordement
+ Echelle de validite
+ Couplage de regime
+ Fonctions Validite / Seuil / Hierarchie
```

Avec une precision :

```text
Raccordement peut etre local, distribue ou attendu.
```

La forme complete est donc :

```text
Architecture effective basse energie =
{G_F, M_W, alpha_s(Q^2), Lambda_QCD, alpha_G(E), M_Pl}
+ {Validite, Seuil, Hierarchie}
+ {regimes de description adaptes}
```

### 13. Ce que le cycle ne dit pas

Le cycle ne dit pas que toutes les constantes effectives appartiennent a cette architecture.

Il ne dit pas non plus que toute constante dependante d'une echelle est automatiquement "effective basse energie".

Il ne transforme pas :

```text
effectif
```

en famille superieure.

Il ne resout pas :

```text
la completion quantique de la gravitation
l'origine des constantes
la valeur profonde des couplages
la nature ultime des seuils
```

Il clarifie seulement les statuts methodologiques des grandeurs dans des regimes ou une description effective devient operatoire.

### 14. Risques controles par le cycle

Le cycle controle quatre risques.

Premier risque :

```text
reduire l'effectivite a la theorie de Fermi
```

Correction :

```text
QCD et gravitation montrent d'autres formes d'effectivite.
```

Deuxieme risque :

```text
confondre seuil et famille
```

Correction :

```text
Seuil reste une fonction transversale.
```

Troisieme risque :

```text
traiter G_N comme un couplage sans dimension comparable a alpha
```

Correction :

```text
le bon objet est alpha_G(E), alpha_G(m), ou un autre rapport sans dimension.
```

Quatrieme risque :

```text
faire de "basse energie" une notion uniforme
```

Correction :

```text
le sens de basse energie depend du secteur physique et du type de description.
```

### 16. Decision taxonomique

La decision finale du cycle est :

```text
Architecture effective basse energie : confirmee comme architecture inter-familles.
```

Statut :

```text
robuste, mais encore ouverte dans ses extensions.
```

Elle est robuste parce que les quatre cas ne forcent pas la meme structure. Ils montrent au contraire que l'architecture peut accueillir plusieurs formes d'effectivite sans les confondre.

Elle reste ouverte parce que d'autres domaines pourraient la tester :

```text
theories effectives nucleaires
EFT du secteur electrofaible au-dela du modele standard
theories effectives cosmologiques
limites de validite en regime de courbure forte
```

Mais ces extensions ne doivent pas etre ajoutees avant consolidation.

### 17. Effet sur la carte generale

Les cartes consolidees actives integrent le resultat suivant :

```text
Architecture effective basse energie =
architecture inter-familles confirmee par le cycle GF / MW / alpha_s / Lambda_QCD / alpha_G / M_Pl.
```

Elles doivent maintenir les precisions suivantes :

```text
1. Raccordement peut etre local ou architectural.
2. Validite devient une fonction transversale centrale.
3. Seuil reste une fonction transversale, non une famille.
4. Hierarchie reste une fonction transversale, non une famille.
5. "Effectif" ne doit pas devenir une famille superieure.
```

Cette integration renforce la distinction :

```text
famille = fonction principale d'une grandeur
architecture = articulation de plusieurs familles dans un secteur ou un regime
```

### 18. Formule issue du cycle

La formule de travail issue du cycle est :

> Une constante effective basse energie n'est pas seulement une valeur stable dans une approximation ; c'est une grandeur qui rend operatoire un regime de description en indiquant ce qui est condense, non resolu, courant, non perturbatif ou supprime.

Formule plus courte :

> L'effectivite stabilise moins une valeur qu'une validite de description.
